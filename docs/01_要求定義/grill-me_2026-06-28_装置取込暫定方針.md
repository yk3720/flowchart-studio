# grill-me — 装置取込 暫定方針（2026-06-28 第2回）

**種別:** grill-me フォローアップ（費用・作者 UX 再検討 · **実装反映済**）  
**正本（技術）:** [ADR-019 暫定方針](<../03_技術仕様/意思決定記録(ADR).md#adr-019-fastapi-装置-excel-ブラウザ取込草案--2026-06-28>) · [Excel取込.md §6](../03_技術仕様/Excel取込.md)  
**前提:** [grill-me 第1回（FastAPI 追加合意）](./grill-me_2026-06-28_FastAPI装置Excel取込.md) · FastAPI 骨格 · UI/プレビュー実装後  
**更新:** 2026-06-28（資料横断整合 · 矛盾解消）

---

## 背景

第1回 grill で **FastAPI + Web xlsx 取込** を合意し、リポ内に API 骨格 · Next プロキシ ·「装置を取込…」UI を実装した。  
その後 **Railway 等の月額コスト** · **作者 PC に Python が無い可能性** · **全作者が各自で取込する運用** を再検討し、**当面は Web を import.json のみ**に戻す方針で合意した。

---

## 1 行決定

**Web「装置を取込…」は import.json のみ。** xlsx 正規化は **作者 PC（CLI → 将来 exe 配布）**。**FastAPI/Railway 本番デプロイは保留**（コードはリポ内に残置）。

---

## 決まったこと

| #   | 論点                    | 決定                                                                                  | 理由（一行）                               |
| --- | ----------------------- | ------------------------------------------------------------------------------------- | ------------------------------------------ |
| 1   | Web 装置取込の形式      | **import.json のみ**                                                                  | 月額回避 · 正規化を作者 PC に集約          |
| 2   | xlsx 正規化の場所       | **作者 PC** — 開発者は `npm run excel:*:normalize` · 作者向けは **変換 exe 配布予定** | Python 未インストール作者を想定            |
| 3   | FastAPI / Railway       | **本番デプロイ保留** — リポ内骨格は **削除しない**                                    | 再開時のコスト削減 · 第1回合意の温存       |
| 4   | Vercel 本番 env         | **`FASTAPI_*` は不要**（JSON のみ期間）                                               | 未デプロイ API を呼ばない                  |
| 5   | 作者フロー              | Excel 編集 → PC 変換 → Web プレビュー → 取込                                          | ドラッグ&ドロップ不要 · ファイル選択で十分 |
| 6   | 取込前 UI               | **プレビュー →「取込」**（第1回合意を維持）                                           | エラー不可 · 警告可 · batch_id 整合        |
| 7   | メニュー名              | **「装置を取込…」**（import.json 専用）                                               | 第1回の入口統合名を維持                    |
| 8   | 表ペイン 1 フロー Excel | **現状維持**（`Excelから取込…` · `parseExcelBuffer`）                                 | 装置 bundle 取込と混同しない               |

---

## 作者フロー（当面）

```text
(1) Excel で装置一式を編集（v0.3 入力用 xlsx）
(2) PC で正規化
      · 開発者: npm run excel:a0001:normalize 等
      · 作者（予定）: 配布 exe で xlsx → import.json
(3) Web ログイン → 装置 workspace → その他 →「装置を取込…」
(4) import.json を選択 → プレビュー（ユニット/フロー数 · エラー/警告）
(5)「取込」確定 → importEquipmentBundle → DB
(6) 左ナビで確認
```

**xlsx を Web に選んだ場合:** 変換手順の案内バナーを表示（FastAPI は呼ばない）。

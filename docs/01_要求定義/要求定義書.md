## 1. プロジェクト概要

**プロジェクト名:** Flowchart Studio（リポ: `flowchart-studio`）  
**立ち上げ日:** 2026-05-20

### 解決する課題

業務フローや手順を**フローチャート**で表現したいが、図形を1つずつ手で並べるのが手間。表形式で構造化したデータから**一貫したレイアウト**で図にしたい。**ブラウザだけ**で完結させたい。

### プロダクトの一文

**表（または同等データ）からフローチャートを自動生成し、ブラウザで確認・保存・画像出力できる Web アプリ。**

### やること / やらないこと

| やる                                              | やらない（本プロジェクト）              |
| ------------------------------------------------- | --------------------------------------- |
| 表 → ノード IR → 自動レイアウト → 画面表示        | Excel への図形描画（COM）               |
| ノード種（端子・処理・判断・入出力・手動入力）    | Office Add-in                           |
| JSON 保存・再読込 · **Supabase クラウド保存**     | Figma 級の自由描画キャンバス            |
| **Vercel 本番公開** · パスワード/OAuth ログイン   | デスクトップ exe の置き換えを約束しない |
| PNG / SVG エクスポート · Excel 正規化パイプライン |                                         |

**注意:** デスクトップ版 MZ0000 は **参照しない**。本プロジェクトは **Web 専用**。

## 2. MVP 定義（凍結 · ADR-005）

**対象:** Phase 1 — **2026-05-20 時点で AC-1〜8 達成**。AC は凍結。

### MVP の一文

**表データからフローチャートを自動生成し、ブラウザで表示し、JSON 保存と PNG 出力ができる。**

### 受け入れ条件

| #    | 条件                                                |
| ---- | --------------------------------------------------- |
| AC-1 | 8列（または7/6列）形式の表を **JSON** で読み込める  |
| AC-2 | 5種ノードが区別して表示される                       |
| AC-3 | 下方向・右方向の接続が矢印で表示される              |
| AC-4 | 判断ノードからの分岐に Yes / No ラベルがある        |
| AC-5 | **自動レイアウト**（表変更 → 再生成で位置が決まる） |
| AC-6 | 図を **JSON** でダウンロード・読み込みできる        |
| AC-7 | 図を **PNG** でダウンロードできる                   |
| AC-8 | 1 画面で「表＋プレビュー」が完結する                |

Post-MVP の実用拡張は [ADR-009](<../03_技術仕様/意思決定記録(ADR).md>)。UI 詳細は [UI仕様.md](../02_機能設計/UI仕様.md)。

## 4. 優先順位（製品戦略 · 2026-05-30 合意）

| 優先         | 内容                                                                        |
| ------------ | --------------------------------------------------------------------------- |
| **最優先**   | フローチャートを**素早く作れる**こと                                        |
| **次点**     | 作ったフローを**簡単に見つけて開ける**こと（DB 保存）                       |
| **後回し可** | 見た目（テーマ・サイズ等）· 年間テーマの本番投入 · ログイン整備（OAuth 等） |

### 検証の方向

- **Web 版で本当にいいか** · **React Flow でいいか**を、触りながら検証
- 年間テーマは急がない。**方向性を決める時間**を使う
- 上司・同僚に見せる前に、**画面・機能をブラッシュアップ**

### 入力・作成の想定

| 項目         | 内容                                                     |
| ------------ | -------------------------------------------------------- |
| 表の作成場所 | **Excel またはスプレッドシート**                         |
| 取込方法     | **コピー貼り付け** · **Excel ファイル読込**              |
| Excel 優先   | **今ある機能を分かりやすくする**（新規開発より UI 整理） |

### 画面整理

- **いらないボタン・表示を減らしてすっきり**させる
- 何を消すかは改めて本人が指定（相談しながら決定）

### 年度末の目標

- **2026 年度中**に、**装置 1 件分のデータが入力できる体制**ができていれば OK

## 5. 手間削減シナリオ（Excel · Web）

### 現状の最短フロー（editor）

| #   | 操作                                                             | 画面           |
| --- | ---------------------------------------------------------------- | -------------- |
| 1   | `npm run excel:new-device -- Z00002 装置名` で入力用 xlsx を生成 | ローカル       |
| 2   | Excel で表を編集 · `npm run excel:normalize` で `import.json` 化 | ローカル       |
| 3   | Web を開く → 左ナビで装置を選ぶ                                  | `/`            |
| 4   | **その他 → import.json を取込…**                                 | ワークスペース |
| 5   | ナビで **動作** を選ぶ → 表を確認 · 必要なら編集                 | 表ペイン       |
| 6   | **再生成** → プレビュー確認                                      | プレビュー     |
| 7   | **その他 → PNG を保存**                                          | 出力           |

**ステップ数目標:** 新規装置 約 7 ステップ（うち Web 内 4）

**権限:** フロー**編集**は editor 同士で可 · **削除**は所有者または admin のみ（DB: 016）

### 年度テーマ用デモ

| 時期     | やること                                          |
| -------- | ------------------------------------------------- |
| 案件選定 | 制御着手前の 1 装置を選ぶ                         |
| 表作成   | 上記 Excel フローで 1 装置分の import.json を用意 |
| レビュー | 関係者に Web URL + PNG を見せる                   |

**未スコープ（要 ADR）:** PLC コメント → 8 列自動変換

### やらないこと（このシナリオの範囲外）

- OAuth 整備 · 本番 Supabase 分離
- Mermaid 版との一本化判断（別途）
- Web で入力用 xlsx を直接パース

## 背景

本番 / 技術デモ / 一般デモの 3 URL 分離後、実機利用で次が判明した。

- 「その他」の **雛形・例** が Nav / 同梱シードと役割重複
- 表下 `<details>`（設計メモ · 回覧 · CSV · 列の意味）が増え、**表が長いと開きにくい**
- 一般デモで Nav ラベルと表内容の不一致（カレー ↔ 朝の出勤）
- デモで **未保存切替確認** が探索を妨げる
- 技術デモのプレースホルダ（DEMO-001 プレス機）では説得力不足
- **主役はフロー図** だが **10 列表は横幅が要る** — 現行 Nav｜表｜図 では意図と配分がずれる

---

## 決まったこと

### 1. 機能整理（図1 — その他 · 雛形）

| 項目               | 決定                                                                             |
| ------------------ | -------------------------------------------------------------------------------- |
| 雛形・例（M1〜M5） | **本番 · デモとも削除**                                                          |
| 代替               | 本番: 装置 Nav + Excel / import.json · デモ: **各動作への同梱シード** + Nav 選択 |

### 2. 3ペイン入替 · 配分（図2 · レイアウト）

| 項目         | 決定                                                                      |
| ------------ | ------------------------------------------------------------------------- |
| 並び         | **Nav ｜ 図（真ん中）｜ 表（右）**（現行 Nav｜表｜図 から入替）           |
| 12 カラム    | **2 ｜ 4 ｜ 6**（全体 **17% ｜ 33% ｜ 50%**）                             |
| 主役 vs 配分 | **主役＝図**（視線 · 意味）· **配分多め＝表**（10 列 · 横スクロール抑制） |
| 現行との距離 | 現行 ≒ 2｜5｜5 — **C 案が最も近い**入替                                   |
| 再考         | 実機で問題あれば比率を調整（ユーザー合意）                                |

**配分の意味（1280px 目安）**

| ペイン | カラム | 幅目安                                     |
| ------ | ------ | ------------------------------------------ |
| Nav    | 2      | ~213px                                     |
| 図     | 4      | ~427px（縦長フロー向け · min ~280px 確保） |
| 表     | 6      | ~640px（現行表 ~545px より広い）           |

### 3. 設計メモ · 表タブ下 · その他（図2）

| 項目             | 決定                                                                         |
| ---------------- | ---------------------------------------------------------------------------- |
| 設計メモ         | 右ペイン **`[表] [設計メモ]` タブ** — 図は常時真ん中、メモ執筆時は右のみメモ |
| 列の意味         | **「その他 ▼」** へ移動（モーダル or 固定幅パネル）                          |
| 表タブ下         | **▶ 確認（警告）· ▶ 回覧** のみ残す                                          |
| CSV / Excel 取込 | **「その他 ▼」** へ移動（表タブから除去）                                    |

**その他メニュー指針（Web 調査 + 合意）**

- 厳密な件数上限はない — **セクション分け · 低頻度操作 · 主操作はヘッダー外**
- 整理後（編集者 · 本番）: 約 **13 項目 / 6 セクション**（雛形・例 −5 · 列の意味 +1 · CSV +1）
- 1 セクション **5 項目以内** を目安（書き出しが 5 で上限）

### 4. 表編集の役割分担

| 項目           | 決定                               |
| -------------- | ---------------------------------- |
| 本格編集       | **Excel 主**（作者正本）           |
| Web 表         | 軽修正 · 取込後確認 · 再生成ループ |
| 図折りたたみ等 | **今回は未実装** — 必要時に追加    |

### 5. 一般デモ（図3）

| 項目       | 決定                                                          |
| ---------- | ------------------------------------------------------------- |
| 階層       | **2 ユニット × 2 動作 ＝ 4 パターン**                         |
| ユニット   | **料理** · **掃除**                                           |
| 動作       | 料理: **カレー** · **パスタ** / 掃除: **キッチン** · **浴室** |
| 同梱シード | 各動作 UUID に初期 JSON（**ひな形メニュー削除とセット**）     |
| 既知不具合 | 「カレー」に朝の出勤内容 → シード + キャッシュ整理で解消      |

### 6. デモ UX（図4）

| 項目         | 決定                                                        |
| ------------ | ----------------------------------------------------------- |
| 未保存切替   | **`AUTH_DISABLED` 時: 確認ダイアログなし · サイレント破棄** |
| 未保存バナー | **非表示**（同上）                                          |

### 7. 技術デモ（図5 · 6）

| 項目   | 決定                                                        |
| ------ | ----------------------------------------------------------- |
| 装置   | **DEMO0000：塗布装置 のみ**（DEMO-001/002 プレス機は削除）  |
| 中身   | **A0001 全量**（5U · 記入済みフロー）— `import.json` ベース |
| 重さ   | 運用して重ければ削る（ユーザー合意）                        |
| 装置増 | 選択肢が増えるだけ — **装置内のユニット/動作ツリー** が重要 |

### 8. 任意案 — デモ限定 UI

| 項目            | 決定                                                            |
| --------------- | --------------------------------------------------------------- |
| 設計メモ · 回覧 | **`AUTH_DISABLED` 時非表示**（設計メモタブ含む）                |
| その他 ▼        | **書き出し（PNG/SVG 等）+ 列の意味 + CSV/Excel 取込** のみ      |
| 非表示          | import.json · JSON 読込/書出 · 危険 · オフライン 等             |
| モバイルタブ    | **図｜表** · **デフォルト図**（デスクトップ「主役＝図」と整合） |

**ADR-017 との関係:** 「一般/技術 UI 同一」は **骨格（3 ペイン · ヘッダー主操作）** を指す。**`AUTH_DISABLED` 時のメニュー/パネル差** は本 grill で明示的に許容。

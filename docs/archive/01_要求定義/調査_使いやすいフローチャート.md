## 1. プロダクトの型

| 型             | 例                           | 本アプリ                               |
| -------------- | ---------------------------- | -------------------------------------- |
| 自由描画       | draw.io、Lucidchart          | 対象外（Figma 級キャンバスはやらない） |
| **データ駆動** | 表→自動生成、テキスト→AI生成 | **ここに集中**                         |

**競合軸:** キャンバス操作の巧みさではなく、**入力の楽さ・再生成の信頼性・出力の実用性**。

---

## 2. 使いやすさの要件（採用するもの）

### 入力

- 表 UI・**CSV 貼り付け**（スプレッドシート入口）
- 列の意味が分かるヘルプ（専門列名だけにしない）
- 雛形・サンプルでゼロから書かない

### 図

- 5 種ノードが一目で区別できる
- 判断は Yes/No（または将来 locale で はい/いいえ）
- 自動レイアウト + ズーム・fitView
- キャンバスは**閲覧専用**と明示（表が SoR）

### フィードバック

- **再生成契約**を貫く（変更 → 再生成 → プレビュー）
- stale 時は **PNG 出力をブロック**
- エラーは**行・セル**へ戻れる（ハイライト・ジャンプ）
- 生成失敗時も**直前の正しいプレビューを維持**

### 運用

- JSON / PNG（SVG は後）
- **localStorage 下書き**（単一ユーザーでも必須級）
- 完成チェックリストで「実務 1 本」まで通す

### 後回し

- 共同編集、認証、クラウド DB
- キャンバス上のドラッグ編集（完成後の拡張候補）

## 1. このセッションの目的

DB-1（`flow_documents` の text `module_id`）から **DB-2** へ移行するため、装置階層テーブル設計 · フロー表の保存形 · Excel 往復 · 削除/viewer 方針を **1 問ずつ** 確定する。

---

## 2. 現状の整理（2026-05-31 時点）

| 項目           | 状態                                                                        |
| -------------- | --------------------------------------------------------------------------- |
| DB-1           | `profiles` + `flow_documents`（`module_id text` · `payload jsonb`）· 実装済 |
| Nav 装置階層   | `moduleHierarchy.ts` 固定（デモ press-01/02）                               |
| 3 ペイン UX    | 実装済（mock 永続化 · ADR-011）                                             |
| 本セッション後 | SQL 003/004 草案 · ADR-014 Accepted（設計）· **マイグレーション未適用**     |

---

## 3. 決まったこと

| 論点              | 決定                                                               |
| ----------------- | ------------------------------------------------------------------ |
| テーブル分離      | 装置 **4 表** + `flow_documents`（jsonb）                          |
| 4 表              | `equipment_codes` → `devices` → `units` → `modules`                |
| 社内番号          | **番号のみ** · **1 社内番号 = 1 装置**                             |
| ユニット/動作キー | Excel 上は **名称** · DB 内部は **uuid**                           |
| フロー保存        | **A:** 1 動作 = `flow_documents` 1 行 · `payload` 丸ごと           |
| Web 編集          | **フロー表の中身のみ** UPDATE（工程行削除含む）                    |
| editor DELETE     | 装置/ユニット/動作/フロー行の **DELETE 禁止** · FK **RESTRICT**    |
| 誤登録削除        | **Supabase プロジェクト管理者** → 最終 **`admin` 画面**（M-1→M-3） |
| `legacy_key`      | **`{社内番号}:{module_slug}`** · 左は必ず社内番号                  |
| デモ seed         | `DEMO-001` / `DEMO-002`（旧 `press-01` / `press-02`）              |
| viewer            | **全閲覧可** · JSON DL 不可は **UI のみ** · 画面変更なし           |
| Excel             | 試作は **後回し** · 方向: 1 ファイル=1 社内番号 · Python 正規化    |

---

## 4. 却下した案

| 案                                    | 却下理由                                   |
| ------------------------------------- | ------------------------------------------ |
| フロー行の DB 正規化（1 行=1 ノード） | Excel/Web 往復が重い · 横断検索要件なし    |
| editor による装置/ユニット DELETE     | 日常不要 · 誤操作リスク                    |
| Web 内で横並び Excel を直接パース     | 原因追跡が難しい · Python+取込用シート方針 |
| フロー行正規化テーブル（B）           | 上記と同型                                 |
| Excel VBA 正規化                      | メンテ負荷 · Python 方針                   |

## 6. grill-me セッション記録（Q&A 要約）

| #   | 論点                            | 回答・結論                                                                              |
| --- | ------------------------------- | --------------------------------------------------------------------------------------- |
| 1   | 装置構成とフロー表              | **テーブル分離** — 将来別アプリが装置 4 表だけ参照可                                    |
| 2   | 装置側の形                      | **4 テーブル:** 社内番号 → 装置（表示名）→ ユニット → モジュール                        |
| 3   | 1 社内番号と装置                | **1:1** — フォルダ管理（1 フォルダ=1 番号）と一致                                       |
| 4   | 社内番号マスタに載せるもの      | **番号のみ** — 客先名等は載せない                                                       |
| 5   | ユニット/動作のキー（人間向け） | **名称** — 社内サブ番号なし                                                             |
| 6   | 裏側 ID                         | **uuid** — Excel には名前だけ · 改名時の不整合を防ぐ                                    |
| 7   | フロー表の保存                  | **A 確定:** `payload jsonb` に `FlowchartDocument` 丸ごと · 行正規化は却下              |
| 8   | Excel ファイル単位              | **暫定 A 最有力:** 1 ファイル = 1 社内番号 — **未確定**（試作で確定）                   |
| 9   | Excel シート単位                | **議論:** 1 シート=1 動作が Web と 1:1 で揃いやすい — **未確定**                        |
| 10  | 正規化の場所                    | **Python 等の別ツール** — Excel マクロ/VBA は使わない                                   |
| 11  | Web で編集する範囲              | **フロー表の中身**（9 列 · 工程行の増減）のみ UPDATE                                    |
| 12  | 装置/ユニット/動作の DELETE     | **editor 日常 UI なし** · RLS DELETE ポリシーなし · FK RESTRICT                         |
| 13  | 誤登録装置の救済                | **M-1** Runbook（Dashboard/SQL）→ **M-2** `admin_delete_equipment()` → **M-3** admin UI |
| 14  | `legacy_key` 形式               | **`{internal_code}:{module_slug}`** — 左は必ず社内番号 · 移行用                         |
| 15  | 旧キー変換                      | `press-01:*` · bare slug → `DEMO-001:*`（004 内関数）                                   |
| 16  | viewer の閲覧範囲               | **全フロー・装置構成閲覧可**（ADR-013 踏襲）                                            |
| 17  | viewer と JSON                  | JSON DL 不可は **ボタン非表示（UI）** — RLS/画面レイアウトは editor と同型              |
| 18  | レビュー反映                    | 003 単体は未完成 → **004 で FK 化** · CASCADE ではなく RESTRICT+DELETE 禁止             |

## 四層（実装）

| 層                 | フォルダ                                                        | 載せるもの                                                |
| ------------------ | --------------------------------------------------------------- | --------------------------------------------------------- |
| **フロントエンド** | `app/` · `frontend/src/`                                        | ページ · UI · デモ JSON（`samples/`）                     |
| **バックエンド**   | `backend/src/` · `app/**/route.ts`                              | Server Actions · 認証 · DB 接続（**別プロセスではない**） |
| **データベース**   | `database/migrations/` · `database/sql/` · `database/src/seed/` | 番号付き DDL · 手動 Run SQL · seed TS                     |
| **Python**         | `python/src/` · `python/scripts/` · `python/testdata/fixtures/` | 正規化ライブラリ · npm 入口 · pytest 用 fixture のみ      |
| **作者データ**     | `data/devices/`                                                 | 装置ごと xlsx（Git 外）· import.json（Git 内）            |
| **共有**           | `lib/`                                                          | 純粋関数（表パース · レイアウト · 型）                    |

> `app/` は Next.js の制約でルート固定。ページ UI も `app/auth/*/route.ts` の API 的処理もここに残る。

### `app/` と `frontend/src/` の境界（2026-06-24 合意）

| 置く場所                   | 載せるもの                                                              |
| -------------------------- | ----------------------------------------------------------------------- |
| `app/`                     | ルート定義のみ — `page.tsx` · `layout.tsx` · `route.ts` · `globals.css` |
| `frontend/src/components/` | すべての UI コンポーネント（`'use client'` 含む）                       |
| `frontend/src/samples/`    | デモ JSON                                                               |

`app/*/page.tsx` は Server Component として認証 · データ取得 · redirect を担い、UI は `@/components/*` を import する。開発用スタイルガイド（`StyleGuideClient`）も `frontend/src/components/dev/` に置く。

### `backend/src/lib/` の区分（2026-06-24 合意）

| フォルダ                       | 載せるもの                                                          |
| ------------------------------ | ------------------------------------------------------------------- |
| `auth/`                        | セッション · ロール · 型 · 認証系 Server Action（`signInPassword`） |
| `supabase/`                    | 接続クライアント · env · middleware · E2E stub                      |
| `flowchart/actions/documents/` | フロー中身の読み書き · reset · 共通ガード（`flowDocumentsAuth`）    |
| `flowchart/actions/hierarchy/` | 装置ツリー参照                                                      |
| `flowchart/actions/delete/`    | 装置 · ユニット · モジュール削除                                    |
| `flowchart/actions/import/`    | 一括取込                                                            |

管理画面（`/admin`）も `@/lib/flowchart/actions/delete/*` 等を直接 import する。re-export 専用の `admin/` フォルダは置かない。

### `lib/flowchart/` の区分（2026-06-24 合意 · C+）

| サブフォルダ | 載せるもの                                                                    |
| ------------ | ----------------------------------------------------------------------------- |
| `model/`     | 型 · スキーマ · バリデーション · `document`                                   |
| `equipment/` | 装置階層 · `DEMO_DEVICES` · `starterFlowSnapshot`                             |
| `table/`     | 列定義 · CSV/Excel/表パース · 部分貼り付け                                    |
| `graph/`     | `generate` · layout · `toReactFlow` · `measureHeights` · `edgeLabelPlacement` |
| `visual/`    | `flowColors` · `statusBanner` · `flowPreviewA11y`                             |
| `policy/`    | 権限 pure 関数（`*DeletePermissions` · `flowResetPermissions`）               |
| `import/`    | 取込 JSON スキーマ · `prepareImportBundle`                                    |
| `browser/`   | `localStorage` / IndexedDB 永続化                                             |

| 層外（ずれていたもの）                | 移動先                                       |
| ------------------------------------- | -------------------------------------------- |
| `moduleDraftLoader`（`"use client"`） | `frontend/src/lib/` · import は `@/client/*` |
| `mapDevicesForClient`（Server DTO）   | `backend/src/lib/flowchart/`                 |

`lib/utils.ts` はルート `lib/` に残す（`cn()` 等）。

### `database/` の区分（2026-06-24 合意 · 案 B）

| フォルダ      | 載せるもの                                                           |
| ------------- | -------------------------------------------------------------------- |
| `migrations/` | 番号付き · append-only · DDL / RLS / RPC（**検証クエリは置かない**） |
| `sql/dev/`    | Dashboard 向け一括 Run · dev 固有データ（`apply_006_*` 等）          |
| `sql/seed/`   | seed 専用 RPC（`seed_equipment_bundle_fn`）· TS 生成の `*_run.sql`   |
| `sql/verify/` | 適用後の読取専用チェック（`verify_db2.sql`）                         |
| `src/seed/`   | `npm run seed:*` · import.json → Supabase                            |

`sql/seed/*_run.sql` は `npm run seed:* -- --write-sql` の出力。SQL Editor 手動 Run 用としてコミット可（TS 再生成で上書き）。

### `python/` の区分（2026-06-24 合意 · 案 B）

| フォルダ               | 載せるもの                                                   |
| ---------------------- | ------------------------------------------------------------ |
| `src/excel_normalize/` | import 可能な正規化ライブラリ                                |
| `scripts/`             | `npm run excel:*` 入口（装置固有 `build_*` 含む · フラット） |
| `tests/`               | pytest（`src` 外）                                           |
| `testdata/fixtures/`   | 装置に紐づかない共有 JSON · 生成 xlsx（Z00001 等）           |
| `templates/`（生成）   | 空テンプレ xlsx · Git 外（`build_template.py`）              |

### `data/` の区分（2026-06-25 合意 · 移行予定）

| フォルダ        | 載せるもの                                                 |
| --------------- | ---------------------------------------------------------- |
| `data/devices/` | 作者ワークスペース（xlsx Git 外 · `import.json` コミット） |

> **移行中:** コード参照はまだ `python/testdata/devices/`。次セッションで `data/devices/` へ移行 · `DEVICES_ROOT` 集約。

## 設計原則

| 層                 | フォルダ                   | 載せるもの                                       |
| ------------------ | -------------------------- | ------------------------------------------------ |
| **憲法**           | ルート `AGENTS.md`         | AI 境界 · 再開手順 · SSOT マップ                 |
| **ドキュメント**   | `docs/`                    | 各フォルダに目次 MD + 必要時のみ個別ファイル     |
| **サンプルデータ** | `frontend/src/samples/`    | アプリ用 JSON（雛形・デモ表）                    |
| **作者データ**     | `data/devices/`            | 装置 xlsx（Git 外）· import.json（Git 内）       |
| **起動**           | `scripts/dev/`             | `open.bat` のみ                                  |
| **Python**         | `python/`                  | Excel 取込パイプライン（`src/excel_normalize/`） |
| **E2E**            | `e2e/`                     | Playwright                                       |
| **セッション**     | `yk-memo/handoffs/{slug}/` | 次の 1 件（**リポ外**）                          |

**MUST NOT:** `specs/` へのリダイレクト stub · `fixtures` を複数系統で使う · `client/` / `server/` 命名（本リポは `frontend/` / `backend/`）

## import エイリアス（`tsconfig.json`）

物理パスと import を両立するため、以下のエイリアスを維持する。

| import                                | 実パス                                          |
| ------------------------------------- | ----------------------------------------------- |
| `@/components/*`                      | `frontend/src/components/*`                     |
| `@/samples/*`                         | `frontend/src/samples/*`                        |
| `@/lib/auth/*`                        | `backend/src/lib/auth/*`                        |
| `@/lib/supabase/*`                    | `backend/src/lib/supabase/*`                    |
| `@/lib/flowchart/actions/*`           | `backend/src/lib/flowchart/actions/*`           |
| `@/lib/flowchart/mapDevicesForClient` | `backend/src/lib/flowchart/mapDevicesForClient` |
| `@/client/*`                          | `frontend/src/lib/*`                            |
| `@/lib/flowchart/*`                   | `lib/flowchart/*`                               |

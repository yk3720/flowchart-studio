# ワークスペースレイアウト方針 — 3ペインリサイズ · ナビ配色

**状態:** 合意済（2026-06-25）· **実装:** PR-B 実装済（2026-06-25）  
**関連:** [UI仕様.md](./UI仕様.md) · [表ペインUX方針.md](./表ペインUX方針.md) · [ADR-016](<../03_技術仕様/意思決定記録(ADR).md#adr-016-10-列順-v2--3-ペインリサイズ2026-06-25>) · [design-system.md](../design-system.md)

---

## 背景

| 現象                   | 原因                                               |
| ---------------------- | -------------------------------------------------- |
| 表ペインを広くできない | 表:図 = `2fr:3fr` 固定（`FC_WORKSPACE_MAIN_GRID`） |
| ナビ幅を変えられない   | `lg:w-[min(20%,240px)]` 固定                       |
| 3領域が同系色で平坦    | ナビ・表・図すべて `flow-surface-muted` 系         |

**方針決定（ユーザー合意）:** 段階導入は行わず、**最初からナビ｜表｜図の3ペインすべて**をドラッグリサイズ可能にする。一度の変更で総コストを最小化する。

---

## 決定事項

### A. 3ペインリサイズ

| 論点            | 決定                                                                                                       | 理由                                                                                                |
| --------------- | ---------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------- |
| 方式            | **react-resizable-panels v4**（`Group` / `Panel` / `Separator`）                                           | 業界標準 · `useDefaultLayout({id})` で永続化 · Next.js SSR 対応。v3 以前と API 命名が異なる点に注意 |
| 対象            | **ナビ ｜ 表 ｜ 図** の3本スプリッター                                                                     | 表↔図だけ後追いすると `FlowchartWorkspace` を二度触る                                               |
| 折りたたみナビ  | 既存 `navCollapsed` を **Panel `collapsible`** と統合検討                                                  | 二重の開閉ロジックを避ける                                                                          |
| モバイル        | **lg 未満は現行どおり** タブ「表｜図」+ ナビはドロワー/折りたたみ                                          | リサイズはデスクトップ専用                                                                          |
| 永続化          | `localStorage` · 外側: `flowchart-studio:workspace-outer-v1` · 内側: `flowchart-studio:workspace-inner-v1` | `useDefaultLayout({id})` フック経由                                                                 |
| 初期比率        | ナビ **18%** · 表 **40%** · 図 **60%**（表｜図は内側 Group で独立管理）                                    | 現行（~20% / 2:3 of 残り）に近い                                                                    |
| 下限（minSize） | ナビ **160px** · 表 **400px** · 図 **280px**                                                               | 装置 select · 10列表 · RF 最低高を確保                                                              |
| 上限            | ナビ **max 28%** または **320px**（実装時に計測）                                                          | 動作名 truncate とのバランス                                                                        |

### B. ナビ配色（2026-06-25 改定）

**経緯:** 初案はナビを深い青背景にする方針だった。Web 調査の結果、プライマリボタン（`--flow-accent` `#2563eb`）とフロー矢印（同色）と**色相が競合**し CTA の視覚階層が弱くなるため、**提案 A（ニュートラル面維持）** を採用。

| 論点             | 決定                                                                                                               | 理由                                                            |
| ---------------- | ------------------------------------------------------------------------------------------------------------------ | --------------------------------------------------------------- |
| 背景             | **現行どおりニュートラル**（`flow-surface-muted` 系 · 表・図と同系統で可）                                         | 大面積に chromatic 青を載せない（Material / shadcn sidebar 型） |
| ゾーン分離       | **明度・ボーダー・タイポ**でナビを識別（全面の色面に頼らない）                                                     | 3 ペインが同系色でも問題なし（ユーザー合意）                    |
| 選択中モジュール | **左ボーダー `flow-accent`** + `flow-accent-selected-bg` / `flow-accent-selected-text`（現行パターン維持・強化可） | 青は**小面積アクセント**に限定                                  |
| プライマリ CTA   | **`--flow-accent` は再生成等の主操作のみ**（変更なし）                                                             | 明るい表ペイン上で最も彩度が高い要素にする                      |
| 表・図ペイン     | **明るい面維持**                                                                                                   | 編集・閲覧の可読性                                              |
| 却下             | ナビ全面の深い青（`#1e3a5f` 等）                                                                                   | CTA・`FLOW_EDGE_STROKE` と同色 family で階層が曖昧              |

**代替（将来検討のみ）:** ナビを**青ではない**暗いスレート（`#1e293b`）にし、選択だけ青アクセント — chromatic 青の大面積は依然として使わない。

**WCAG:** 選択・focus のコントラストは PR 前に計測。`text-flow-text-muted` を暗い面に載せる場合は 4.5:1 を確認。

### C. ナビ左揃え（既知バグ）

| 論点                 | 決定                                                                                                             |
| -------------------- | ---------------------------------------------------------------------------------------------------------------- |
| ユニット・動作ラベル | `fcNavUnitToggle` / `fcNavModuleBtn` に **`justify-start`** を追加（`fcTargetMin` の `justify-center` を上書き） |

本書のリサイズ実装と**同一 PR または直前 PR** で対応。

## SSR · Next.js 注意

- `Group` に **`defaultLayout`** を渡し hydration shift を軽減（[react-resizable-panels SSR 例](https://react-resizable-panels.vercel.app/examples/persistent-layout/server-rendering)）
- パネル数・構成変更時は **`useDefaultLayout({id})` の `id` をバージョンアップ**（古い localStorage を無効化）
- `FlowchartWorkspace` は `"use client"` のため Group もクライアント内
- v4 では `PanelGroup`→`Group` · `PanelResizeHandle`→`Separator` · `autoSaveId`→`useDefaultLayout` と命名変更（詳細: `REACTFLOW_RULES.md §5.6-10`）

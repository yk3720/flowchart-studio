# プレビュー上部 chrome 方針 — 縦スペース確保

**状態:** 合意済（2026-06-25）· **実装:** handoffs §4 参照（**#8** · 表ペイン UX #7 の後）  
**関連:** [UI仕様.md](./UI仕様.md) · [design-system.md](../design-system.md) · [作者ガイド.md](../03_技術仕様/作者ガイド.md) · [表ペインUX方針.md](./表ペインUX方針.md)（ステータス帯の表列統合は同系 · **実装順は表ペインUX方針を先**）

---

## 背景

- フローは **縦方向に長く**なる。横（表:プレビュー比率）は現状で問題なし。
- プレビュー列の **上面**に chrome が重なり、最初の一画面で見えるフロー行数が減る。
- **閲覧者**はフロー主・表副。将来 **編集者**に成長するため、レイアウト骨格は editor/viewer 共通のままとする。

---

## 現状（2026-06-25 時点）

デスクトップ `lg` 以上で、プレビュー列の上には次が重なる。

| 層  | 配置                 | コンポーネント / class                                   |
| --- | -------------------- | -------------------------------------------------------- |
| ①   | **全幅**             | `AppAuthBar`（`fcAuthBar`）                              |
| ②   | **全幅・条件付き**   | ステータスバナー（`fcStatusBanner`）                     |
| ③   | **プレビュー列のみ** | 凡例 + ズーム帯（`fcPreviewChrome` · `FlowColorLegend`） |
| ④   | プレビュー列         | `FlowCanvas`                                             |

表ペインヘッダー（タイトル · 再生成ツールバー）は **左列**にあり、プレビュー列の上ではない。

---

## 決定事項

| 論点                       | 決定                                                         | 理由                                                   |
| -------------------------- | ------------------------------------------------------------ | ------------------------------------------------------ |
| 認証バーの位置             | **表ペイン列の上のみ**（プレビュー列の上には載せない）       | フロー上面の縦を確保                                   |
| ステータスバナー           | **認証バーと同様・表列の上のみ**                             | 同上                                                   |
| 左ナビの上                 | 認証・ステータスは **載せない**                              | 「表の上まで」の解釈                                   |
| 色凡例                     | **プレビューから削除**                                       | ユーザー合意 · 色はノード表示で足りる                  |
| 色の意味の参照             | 表の列ヘルプ · [作者ガイド.md](../03_技術仕様/作者ガイド.md) | 凡例代替                                               |
| ズーム UI                  | **キャンバス右上に固定オーバーレイ**                         | 帯 1 行分を回収 · 開始ノード（左上寄り）と重なりにくい |
| ペイン比率（2fr:3fr）      | **変更しない**                                               | 横は問題なし                                           |
| editor / viewer レイアウト | **同型のまま**（`readOnly` は操作のみ）                      | 閲覧者→編集者の成長パス                                |

---

## 目標レイアウト（デスクトップ `lg`）

```
┌──────┬─────────────────────────┬───────────────────┐
│      │ ① 認証バー               │                   │
│ Nav  │ ② ステータス（あれば）    │  フロー            │
│      │ ③ 表ペインヘッダー        │  （上端から）      │
│      │ ④ 表本体                 │  [ズーム: 右上]   │
└──────┴─────────────────────────┴───────────────────┘
```

**モバイル:** 表タブ／プレビュータブ切替は維持。認証・ステータスは **表タブ側の上** に置く。

## 実装メモ（エージェント向け）

| 変更                         | 主なファイル                                                                               |
| ---------------------------- | ------------------------------------------------------------------------------------------ |
| ワークスペースグリッド再構成 | `FlowchartWorkspace.tsx`                                                                   |

| ズームオーバーレイ class     | `flowchartUiClasses.ts`（`fcPreviewChrome` 非推奨化または帯削除）                          |
| PNG/SVG 出力                 | ズームは `data-flowchart-export-root` **の外**（兄弟要素）— 現行と同様に画像へ混入させない |
| stale オーバーレイ           | ズームより下の z-index を維持                                                              |
| キーボード                   | `FlowCanvas` の `+` / `−` / `Ctrl+0` は維持（a11y）                                        |

### ズームオーバーレイ要件

- 位置: キャンバス親 `relative` 内 · `absolute top-2 right-2 z-10`
- `pointer-events-auto`（ボタンのみ）· 半透明背景で視認性確保
- `data-testid="flow-zoom-percent"` · `flow-zoom-home` は維持（E2E · a11y）

### 凡例

- `FlowColorLegend` をワークスペースプレビューから **出さない**（StyleGuide は残してよい）
- [作者ガイド.md](../03_技術仕様/作者ガイド.md) の「プレビュー左下凡例」記述を実装時に更新

## 却下した案

| 案                               | 理由                                         |
| -------------------------------- | -------------------------------------------- |
| プレビュー全画面（表ペイン削除） | 閲覧者も表を補足参照する                     |
| ペイン比率の変更                 | 横は問題なし（ユーザー確認）                 |
| ズームを左上固定                 | 開始端子と重なりやすい                       |
| 閲覧者専用レイアウト分岐         | 将来編集者への成長 · DB 設計の同型方針と矛盾 |

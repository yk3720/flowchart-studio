## 背景

> **注:** 以下は **実装前（〜2026-06-24）** の課題記述。2026-06-25 の `523a008` で解消済み。

| 現象                                               | 原因（コード）                                                                                                                     |
| -------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| モジュール選択前後で表ペインの見え方が大きく変わる | 未選択時は空状態のみ。選択後に取込パネル・ヒント・表本体が一括表示                                                                 |
| 横スクロールバーが見つけにくい                     | `FlowchartEditor` 表ペイン body と `fcTableScroll` の **二重 `overflow-auto`**。横バーは内側コンテナ最下部にあり、行が多いと画面外 |
| 長文セルが読めない                                 | `table-layout: auto` + `w-full` で列が均等縮小。セル `input` がクリップ                                                            |

## 決定事項

### A. レイアウト統一（モジュール選択前後）

| 論点                                 | 決定                                                                      | 理由                                                                                      |
| ------------------------------------ | ------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------- |
| CSV / Excel 取込（`CsvPastePanel`）  | **`<details>` デフォルト閉**                                              | 選択後も上端を空状態に近づける。7月デモ用にラベル「CSV / Excel 取込」は常時見える         |
| 「表を編集したあとは…」ヒント        | **取込 `<details>` 内または初回のみ**                                     | 常時1行占有を避ける                                                                       |
| ステータス「クラウドから読み込み」等 | **ワークスペース全幅帯を廃止** → **表ペインヘッダー右ステータス行に統合** | 縦1行を回収（[プレビュー上部chrome方針](./プレビュー上部chrome方針.md) の表列移動と整合） |
| エラー・取込完了・オフライン警告     | ヘッダー統合または **一時表示**（`role="alert"` / `role="status"` 維持）  | 常時の「クラウドから読み込み」だけ非表示化                                                |

### B. 列幅リサイズ + リセット

| 論点           | 決定                                                                                                                          | 理由                                                                                                                                                                     |
| -------------- | ----------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 方式           | **`table-layout: fixed` + `<colgroup>`** + カスタム hook（**新規依存なし**）                                                  | 既存 `onPasteCapture`・行編集 UI を維持。TanStack / React Aria 全面導入は過剰                                                                                            |
| 操作           | ヘッダー右端 **ドラッグハンドル**（8px ヒット領域）                                                                           | Excel / AG Grid 慣行                                                                                                                                                     |
| リセット（UI） | **T4「列幅をリセット」は廃止**（2026-06-25 合意）。列幅はヘッダードラッグ + `localStorage` のみ。明示リセットボタンは持たない | ペイン幅リセット（T4 差し替え）と役割分離 · [ワークスペースレイアウト方針 §E](./ワークスペースレイアウト方針.md#e-デフォルト比率改定--ペイン幅リセット2026-06-25-実装済) |
| デフォルト幅   | `lib/flowchart/table/tableColumnWidths.ts`                                                                                    | 列別 px（ID 狭 · Text 広 等）                                                                                                                                            |
| 長文           | `text-overflow: ellipsis` + `title` ツールチップ。列を広げれば編集時全文                                                      | リサイズとセット                                                                                                                                                         |
| 永続化         | `localStorage`（キー: `moduleId + colCount + schema + v1`）                                                                   | 手戻り許容前提で UX 向上                                                                                                                                                 |
| キーボード     | 列ヘッダーフォーカス → Enter → ←/→ 調整 → Esc/Enter 終了                                                                      | Salesforce UX · React Aria 型（range を Tab 停止にしない）                                                                                                               |

### C. 横スクロールバー常時表示

| 論点           | 決定                                                                                                                          | 理由                                 |
| -------------- | ----------------------------------------------------------------------------------------------------------------------------- | ------------------------------------ |
| CSS のみ       | **不可**（縦長表で横バーを viewport 下端に固定できない — [W3C csswg #2252](https://github.com/w3c/csswg-drafts/issues/2252)） | ユーザー調査で確認                   |
| 採用方式       | **ドック型ネイティブ横スクロールバー** — 表ビューポート下端固定の別 `div` と `scrollLeft` 双方向同期                          | Stack Overflow 定番 · ResizeObserver |
| スクロール責務 | **表内部 1 か所のみ**（親ペイン `overflow-auto` 廃止）                                                                        | 二重スクロール解消                   |
| 横バー表示     | `overflow-x: scroll` + 必要なら `padding-bottom` ガター                                                                       | `scrollbar-gutter` は横非対応（MDN） |
| a11y           | ドック div は **`aria-hidden="true"`**（装飾用同期）。操作は表本体のネイティブスクロールバー                                  | fake scrollbar は使わない            |

> **§E 改定（2026-06-26 実装済）:** 横バーの**画面上の位置**は「ビューポート下端」から **操作ボタン行の下・列ヘッダーの上**（固定ゾーン）へ変更。ドック型同期方式は維持 — [§E 改定パック](#e-表ペイン-ux-改定パック2026-06-26--実装済)。

### D. 表ペイン縦優先レイアウト（2026-06-25 実装済 · §E で一部上書き予定）

**背景:** 表が主役なのに、警告・取込・ヘルプが表より上にあり縦スペースを占有。T4 は表列幅用だが、ユーザー要望は**ペイン幅**リセット。

| 論点                     | 決定                                                                                       | 理由                                                                      |
| ------------------------ | ------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------- |
| 表の位置                 | **ヘッダー直下で表本体を最上段**（`TableScrollViewport` を先に配置）                       | 編集の主役を上に · 10 列の横スクロールを早く見せる                        |
| エラーバナー（B1）       | **表の上**（現行維持）                                                                     | `role="alert"` · 生成停止系は常時見える必要                               |
| 警告バナー（B2）         | **表の下** · **`<details>` デフォルト閉** · summary「確認（警告）」                        | 生成は継続（ADR-009）· 表を圧迫しない · CSV 取込と同型                    |
| 表ツールバー             | **表の下** · `[行を追加]` · `[ペイン幅をリセット]`（T4 差し替え）· `N行·10列`              | 主操作は表の近く · 列幅リセット UI は廃止（§B）· **§E で T1/T4 移動予定** |
| CSV / Excel 取込         | **表の下** · `<details>` デフォルト閉（現行維持）                                          | サブ機能 · 7 月デモの summary ラベルは維持                                |
| 列の意味（ヘルプ）       | **表の下** · `<details>` デフォルト閉（現行維持）                                          | 同上                                                                      |
| T4「列幅をリセット」     | **削除**                                                                                   | 列幅はドラッグ + 永続化で足りる · ペイン幅リセットへ置き換え              |
| T4「ペイン幅をリセット」 | **新設**（表ツールバー）· outer + inner Group をデフォルト比率へ · `localStorage` もクリア | [ワークスペースレイアウト方針 §E](./ワークスペースレイアウト方針.md)      |

**目標レイアウト（§D 適用後 · モジュール選択済 · デスクトップ）**

```
┌─ 表ペインヘッダー（固定）──────────────────────────────┐
│ Flowchart Studio · 文脈 · [生成完了… / クラウド読込]   │
│ [再生成] [その他 ▼]                                     │
├─ 表エリア（flex-1 · min-h-0）──────────────────────────┤
│ （エラー時のみ）赤バナー — 表の上                        │
│ ┌ TableScrollViewport（縦横スクロール唯一）──────────┐ │ ← 主役・最上段
│ │ thead sticky                                      │ │
│ │ tbody … input/select                              │ │
│ └───────────────────────────────────────────────────┘ │
│ ═══════════ ドック型横スクロールバー ═══════════════ │
│ [行を追加] [ペイン幅をリセット]  N行·10列               │
│ ▶ 確認（警告）              ← details・デフォルト閉    │
│ ▶ CSV / Excel 取込          ← details・デフォルト閉    │
│ ▶ 列の意味（ヘルプ）                                    │
└──────────────────────────────────────────────────────┘
```

### E. 表ペイン UX 改定パック（2026-06-26 · 実装済）

**背景:** 実地利用で、(1) 表下のツールバー・横スクロールバーが縦スクロール後に触れにくい、(2) 表ペインヘッダーがスクロールで消える、(3) タイトル行が縦スペースを占有、(4) 種別/色 select で他列の値がずれる、(5) その他メニューがナビに隠れる、が判明。grill-me（2026-06-26）で決定。**実装:** `c92f345`（2026-06-26）· 色列 normalize 追修正 `d2603e5`。

**適用範囲:** **デスクトップ 3 ペインのみ**（`lg` 以上 · `workspaceMode && isDesktop`）。モバイル（表/図タブ）は現行維持。

| 論点                              | 決定                                                                                       | 理由                                                                                                                       |
| --------------------------------- | ------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------- |
| スクロール時の固定範囲            | **文脈行 + 操作ボタン行 + 列ヘッダー行** の 3 段を固定                                     | 編集中に「どの動作か」「どの列か」を常時見える                                                                             |
| 「表」見出し（`h2`）              | **削除**                                                                                   | 文脈行で位置は足りる · 縦 1 行回収                                                                                         |
| 「実用版」バッジ                  | **表ペインから削除**（タイトル移動に伴い）                                                 | 縦スペース回収                                                                                                             |
| アプリタイトル                    | **ナビ先頭**「フロー」→ **「Flowchart Studio」** 全文（truncate 可）                       | [ワークスペースレイアウト方針 §F](./ワークスペースレイアウト方針.md#f-ナビタイトル--ペイン幅リセット移動2026-06-26-実装済) |
| **行を追加**（T1）                | **表ペインヘッダー** · `[再生成]` の右                                                     | 主操作を上段に集約                                                                                                         |
| **ペイン幅をリセット**（T4→N9）   | **ナビ** · 全ユニット折りたたみ（N8）の横 · **アイコンのみ** · `title`/`aria-label` で説明 | レイアウト操作はナビに集約 · 表下から除去                                                                                  |
| 横スクロールバー                  | **操作ボタン行の下 · 列ヘッダーの上**（固定ゾーン内）                                      | 表を縦スクロールしても常に触れる（§C の「ビューポート下端」から変更）                                                      |
| 列幅を内容に合わせる（T5 · 新設） | **横スクロールバー行の右端** · アイコン · 全列一括 · `localStorage` 保存                   | 横スクロールと列幅調整を同一ゾーンに                                                                                       |
| 表下ツールバー                    | **行数メタのみ**（`N行·10列`）— T1/T4 は除去                                               | §D の表下ツールバーから主操作を移す                                                                                        |
| 色列セレクト表示                  | 空値のラベル **「（通常）」→「-」**（**セル値は空のまま**）                                | 列幅節約 · ツールチップで「通常・自動運転」                                                                                |
| 種別/色 select バグ               | **修正済** — 色選択時に他列がずれ色列は変わらない（例: ID10）· v2 normalize `d2603e5`      | Vitest + Playwright 回帰済                                                                                                 |
| その他メニュー                    | **`position: fixed`** · トリガー右下基準 · ナビに隠れない                                  | 現行 `absolute` が左クリップされる                                                                                         |
| 装置を削除（N3）                  | **その他 ▼ · 危険** へ移動（M12）                                                          | 低頻度 · ナビ 1 行節約                                                                                                     |

**§C / §D との関係:** §E 実装後、横バー位置（§C）· 表下 T1/T4（§D）· 表ペイン内タイトル/バッジは **§E が優先**。§D の警告/取込/ヘルプ `<details>` 配置は **維持**。

| 項目              | 内容                                                                                                |

**フォルダ内ルール**

- 正本 `{社内番号}_{装置名}.xlsx`（フォルダ名と同名 · **Git 対象外**）
- **共有は `import.json`**（**Git 対象** · 1 装置 1 ファイル）
- **v0.3**（構成6列 + U0〜U9 + マスター表）
- 1 動作試行 `_scratch/{動作名}.xlsx`（**Git 対象外**）
- 手書き作業中の実装置: **`A0001_塗布装置`**

**将来（実運）:** 会社ルールの装置ごと設計フォルダ（社内サーバー）に xlsx · import.json を置く。Git 管理外。Web 取込はファイル選択。

### 3.2 シート構成

| シート         | 枚数           | 内容                                                   |
| -------------- | -------------- | ------------------------------------------------------ |
| **構成**       | **1**          | 装置階層一覧（左ナビの SSOT）                          |
| **ユニット名** | **ユニット数** | 当該ユニット内の **複数動作** のフロー表（**横並び**） |

**構成シート（4 列 · 1 行目ヘッダー）**

| 列       | 例           | DB                              |
| -------- | ------------ | ------------------------------- |
| 装置製番 | Z00001       | `equipment_codes.internal_code` |
| 装置名   | プレス機 B   | `devices.display_name`          |
| ユニット | 供給ユニット | `units.label`                   |
| 動作     | 取出         | `modules.label`                 |

- 行順 = Nav の並び（`sort_order`）
- **装置製番 = 社内番号**（用語をテンプレで固定）

### 3.3 ユニットシート — 横並びの最小ルール

作者の使い勝手のため **表は横に並べてよい**。正規化の安定性のため、**各動作ブロックを Excel の「テーブル」として登録**する（推奨 · 初版 SSOT）。

| ルール | 内容                                                                                             |
| ------ | ------------------------------------------------------------------------------------------------ |
| **R1** | シート名 = 構成シートの **ユニット列と完全一致**（Excel 31 文字制限に注意）                      |
| **R2** | 各動作 = **1 つの Excel テーブル**（挿入 → テーブル）                                            |
| **R3** | テーブル名 = **`{ユニット短名}_{動作名}`**（例: `供給_取出`）— **ブック全体で一意**（実装 v0.1） |
| **R4** | テーブル内 = **10 列**（[作者ガイド.md](./作者ガイド.md) · `table-10col-v1`）                    |
| **R5** | 結合セルは **テーブル内で使わない**                                                              |

> **却下:** 1 シート内の動作を縦積みのみで書かせる方式（入力しにくい）。

> **Power Query 利用時:** R2（Excel テーブル）が揃っていれば、作者 PC 上で取込用シートを更新できる。出力形式は Python と同一とする。

## 4. Python 正規化（正本）

### 4.1 置き場

`c:/yk-application/flowchart-studio/python/` · [README](c:/yk-application/flowchart-studio/python/README.md)

### 4.2 入出力

**入力:** 入力用 `.xlsx`

**出力:** `import.json`（または `import.xlsx` + `_取込` シート群 — **JSON を第一候補**）

```json
{
  "internal_code": "Z00001",
  "display_name": "プレス機 B",
  "units": [
    {
      "label": "供給ユニット",
      "sort_order": 0,
      "modules": [
        { "label": "取出", "sort_order": 0 }
      ]
    }
  ],
  "flows": [
    {
      "unit_label": "供給ユニット",
      "module_label": "取出",
      "title": "取出",
      "payload": {
        "schema": "table-10col-v1",
        "table": [ ["10", "端子", …] ]
      }
    }
  ]
}
```

### 4.3 処理手順

1. 「構成」シートをパース → `units` / `modules` リスト
2. 各ユニットシートを走査 → **Excel テーブル**（または名前付き範囲）ごとに 10 列データ抽出
3. テーブル名 ↔ `(unit_label, module_label)` を解決
4. **整合性検証**（§5）— 失敗時は非 0 終了 + 作者向けメッセージ
5. `import.json` 出力

### 4.4 テスト

- pytest + 入力用 fixture xlsx（1 装置 · 2 ユニット · 各 2 動作 · 横並び）
- 構成 ↔ フロー不一致 · 列数不足 · 未知テーブル名 をケース化

## 5. 整合性検証（Python · Web 共通）

| #   | 検証                                                             | 失敗時                           |
| --- | ---------------------------------------------------------------- | -------------------------------- |
| V1  | 構成の `(ユニット, 動作)` ごとに `flows` が 1 件                 | **警告**（段階手書き · §11）     |
| V2  | `flows` の `(unit_label, module_label)` が構成に存在             | エラー                           |
| V3  | 各 flow の表が **10 列**（ヘッダー行の有無はパーサー規約で統一） | エラー                           |
| V4  | v0.3: 構成にあるユニットの **U{n} シートが未作成**               | **スキップ**（段階手書き · §11） |
| V5  | 構成に無いユニットシートがある                                   | エラー                           |
| V6  | `internal_code` がファイル内で一意                               | エラー                           |

**警告（取込は続行可）**

- 構成にあるが `flows` が未登録 → 「フロー未登録」警告（`import.json` には `units` のみ · `flows` は記入分だけ）
- ユニットシート（U0 等）が未作成 → 当該ユニットはスキップ（エラーにしない）
- DB に存在するが構成に無い行 → **orphan 警告**（DELETE はしない · ADR-014）

## 6. Web 取込（v0.1 実装済み）

- **読むもの:** `import.json` のみ（Python 正規化出力）
- **入口:** その他メニュー → **「import.jsonを取込…」**（workspace · editor のみ）
- **保存:** Server Action `importEquipmentBundle` → RPC **`import_equipment_bundle`**（`005_import_equipment_bundle.sql`）
- **初版 upsert:** 名称マッチ · `sort_order` 更新 · 新規 INSERT — **prune / rename 非対応**

**dev 適用手順:** `005_import_equipment_bundle.sql` を Supabase Dashboard / CLI で 004 の後に適用。

**Web 表ペイン Excel 取込:** `CsvPastePanel` の **Excelから取込…** は `parseExcelBuffer` 経由で取込後 **`ensureParsedTable10ColV2Order`** により v2 列順へ正規化（`REACTFLOW_RULES` §5.6-2b）。恒久 repair スクリプトは作らない。正本経路は引き続き Python → `import.json`。

## 11. 段階取込ロードマップ（2026-06-25）

**背景:** A0001 は構成・マスター（100 モジュール）を先に定義し、**U0 からフロー表を随時手書き**する。U1〜U9 シートは後から追加する。

**作者運用の原則**

| 層                    | いつ埋める | import.json / DB                     |
| --------------------- | ---------- | ------------------------------------ |
| 構成 · マスター       | 先に全体像 | `units` / `modules` は構成どおり全件 |
| U{n} シートのフロー表 | 随時       | `flows` は **記入済み分だけ** upsert |

### 実装タスク（着手順）

| #     | タスク                | 成果物                                                                | 状態 |
| ----- | --------------------- | --------------------------------------------------------------------- | ---- |
| **2** | **xlsx 中身確認 CLI** | `npm run excel:inspect -- A0001` · `python/scripts/inspect_device.py` | ✅   |
| **1** | **部分正規化**        | 未作成 U{n} スキップ · 未記入フローは警告 · `normalize` 成功          | ✅   |
| **3** | **§5 と実装の整合**   | `validate.py` · `tables.py` · 本 §5 表                                | ✅   |
| **4** | **運用の一行 SSOT**   | `data/devices/README.md` · `AGENTS.md` · `python/README.md`           | ✅   |

### `excel:inspect` の出力（人間向けサマリー）

- シート一覧 · 構成のユニット数 / モジュール数
- **U{n} シート:** あり / なし
- **フロー記入済み:** `(ユニット · モジュール · 行数)` 一覧
- **未記入:** 構成にあるがテーブルなし
- **normalize 可否:** ブロッカー（構成外シート · テーブル名不一致等）の有無
- `--json` でエージェント向け構造化出力

### 取込フロー（段階手書き時）

```text
手書き xlsx
  → npm run excel:inspect -- A0001     # 記入状況確認
  → npm run excel:a0001:normalize      # import.json（flows は記入分）
  → npm run seed:a0001 または Web 取込  # DB upsert
```

> **注意:** 空の U{n} シートだけ作ると「テーブルなし」で正規化が止まる。未着手ユニットは **シート自体を作らない**。

**移行期（A0001）:** 構成のモジュール名を `M000…` に更新したあとも、U0 の Excel テーブル名が旧 `動作000` のままなら **n 番目で対応付け**（`tables.resolve_table_module_label`）。新規はテーブル名 = 構成のモジュール列と一致させる。

**記録日:** 2026-06-06  
**根拠:** Web 調査（JIS X 0121 · ISO 5807 · 製造 HMI · WCAG）· [色列\_仕様整理.md](./色列_仕様整理.md)

## 1. 結論（1行）

**図形5種＋色4種は現状維持で十分。** 迷いを減らすのは種類追加より、下記の**使い分けルール**を守ること。

| 項目                    | 判断                             | 備考                                         |
| ----------------------- | -------------------------------- | -------------------------------------------- |
| 図形5種                 | **維持**                         | JIS X 0121 / ISO 5807 の日常記号をカバー     |
| 色4種（空＋黄・橙・青） | **維持**                         | 赤・緑は追加しない（表UI・色弱・HMI慣行）    |
| 将来候補                | **定義済み処理（サブルーチン）** | 同じ共通シーケンスの繰り返しが面倒になったら |

## 2. 図形種別 — 使い分け

| 図形種別     | 標準上の意味 | 自動機フローでの用途                                   | 例（Text1）                   |
| ------------ | ------------ | ------------------------------------------------------ | ----------------------------- |
| **端子**     | 開始・終了   | 運転開始 / サイクル完了 / 異常終了                     | `運転開始` · `サイクル完了`   |
| **処理**     | 演算・動作   | シリンダ動作 · タイマ待ち · データ書込                 | `シリンダ前進` · `5s待ち`     |
| **判断**     | 条件分岐     | センサON · 位置完了 · レシピ分岐（**Yes=下 · No=右**） | `原点完了?` · `レシピA?`      |
| **入出力**   | データ入出力 | PLC↔上位 · レシピ読込 · ログ出力                       | `レシピ読込` · `完了信号出力` |
| **手動入力** | 人による操作 | 部品セット · 確認ボタン · メンテ                       | `部品セット` · `確認ボタン`   |

**記号を増やさず書く**

- **待ち・タイマ** → **処理** ＋ Text1（例: `3s待ち`）
- **初期化** → **処理** ＋ Text1（例: `初期化`）
- **共通シーケンス呼び出し** → 当面 **処理** ＋ Text1（例: `共通:原点復帰`）。将来サブルーチン記号を検討

**標準にあるが本アプリ未実装（参考）**

- 定義済み処理（サブルーチン）— PLC の JSR / SFC 呼び出し相当
- JIS Z 8206 の検査記号 — **QC 工程表**用（制御シーケンス図とは別系統）

## 3. 色列 — 使い分け（凡例）

JIS/ISO に**フローチャートの色の統一規格はない**。3〜4色＋凡例＋形・ラベル併用がベストプラクティス。

| 色（セル値）   | 意味             | 自動機フローでの使い方                                                                   |
| -------------- | ---------------- | ---------------------------------------------------------------------------------------- |
| **（空）通常** | 標準ステップ     | **自動運転の大部分**（全体の9割以上を目安）                                              |
| **黄**         | 判断・注意の強調 | **重要な分岐・インターロック条件**の判断菱形。Text1 に条件を必ず書く                     |
| **橙**         | 要注意・異常系   | **異常復帰・要注意・解除が必要な工程**。Text1 に理由を書く。**表のエラー行（赤）とは別** |
| **青**         | 手動・確認       | **オペ確認・手動介入・セットアップ**。図形種別「手動入力」と併用推奨                     |

**意図的に入れない色**

| 色  | 理由                                                    |
| --- | ------------------------------------------------------- |
| 赤  | 表 UI のエラー行（赤 ring）と混同。HMI でもアラーム専用 |
| 緑  | 色覚多様性。正常/異常を赤緑だけで伝えない               |

## 4. Yes / No とループ（上へ戻る）

| 分岐    | 表の列         | 図の向き                |
| ------- | -------------- | ----------------------- |
| **Yes** | **接続先(下)** | 下へ進む                |
| **No**  | **接続先(右)** | 右へ出る（ラベルは No） |

**上へ戻るループ（推奨パターン）**

過去の ID を **接続先(右) に直接書かない**。右列（**同じ段 · 列=1**）に **処理** 1 行を置き、そこから **接続先(下)** で戻すと、プレビューでループが読みやすい。

```
判断 ─Yes(下)─→ 次のステップ
  │
  No(右)
  ↓
右列の処理（例: あと5分炒める）
  │
  └─(下)→ 戻り先の処理
```

| 例     | 判断          | No → 右列      | 右列 → 接続先(下)    |
| ------ | ------------- | -------------- | -------------------- |
| カレー | 玉ねぎは透明? | あと5分炒める  | 炒める（40）         |
| カレー | 沸騰した?     | もう少し煮込む | 加水して煮込む（70） |

**正本サンプル:** `flowchart-studio/frontend/src/samples/sample-curry.json` · E2E `e2e/curry-loop.spec.ts`

**やらない:** 判断の No を過去 ID に直結だけで済ませる（矢印が見えにくい）

## 5. 二重コーディング（推奨）

色だけに頼らない（WCAG · High Performance HMI と同趣旨）。

| 伝えたいこと       | 図形         | 色         | ラベル              |
| ------------------ | ------------ | ---------- | ------------------- |
| 重要な分岐         | **判断**     | 黄（任意） | 条件を Text1 に明記 |
| 人の操作           | **手動入力** | 青（推奨） | 作業内容を Text1 に |
| 要注意工程         | **処理** 等  | 橙         | 理由を Text1 に     |
| 通常の自動ステップ | **処理** 等  | （空）     | 動作名を Text1 に   |

---

## 6. やること / やらないこと

**やること**

1. 色は **全体の1割未満** に留める
2. プレビュー左下の **凡例** を参照する（常時表示）
3. 色変更後は **再生成** でプレビューに反映
4. 「5s待ち」「原点復帰」など **特殊動作は Text1 に明記**

**やらないこと**

- ステップごとに意味のないカラフル化
- 赤＝エラー · 緑＝OK だけの配色
- 色だけで異常/正常を区別（ラベルなし）
- 表バリデーションの赤/amber と **色列** を混同しない

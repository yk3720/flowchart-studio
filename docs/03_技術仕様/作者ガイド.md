## 1. 列の意味

> **schema（2026-06-25）:** 新規表・正規化出力の既定は **`table-10col-v2`**（[ADR-016 PR-A](<./意思決定記録(ADR).md#adr-016-10-列順-v2--3-ペインリサイズ2026-06-25>) · `67eff04`）。**色列は図形種別の直後（index 2）**。既存 `table-10col-v1` · 作者 Excel（v1 列順）は読込・取込時に v2 へマイグレーション。

### 10 列 v2（推奨 · 新規の正本）

| #   | 列         | 説明                                                                |
| --- | ---------- | ------------------------------------------------------------------- |
| 1   | ID         | 図形 ID（10, 20, 30…）                                              |
| 2   | 図形種別   | 端子 · 処理 · 判断 · 入出力 · 手動入力                              |
| 3   | 色         | 背景色（§3）。通常=自動運転 · 黄=重要判断 · 橙=要注意 · 青=手動確認 |
| 4   | 接続先(下) | 下方向の接続先 ID                                                   |
| 5   | 接続先(右) | 右方向の接続先 ID                                                   |
| 6   | 段         | レイアウト段（tier）                                                |
| 7   | 列         | レイアウト列（level）                                               |
| 8   | Text1      | 主テキスト（MR 番号 · ステップ名等）                                |
| 9   | Text2      | 補足 1 行目                                                         |
| 10  | Text3      | 補足 2 行目                                                         |

schema: `table-10col-v2` · コード: `TABLE_HEADERS_10_V2` · `flowColors.ts`（色列 = index 2）

### 10 列 v1（後方互換 · 既存 Excel）

列 1〜9 は下記「9 列」と同じ。10 列目に「色」。

| 列  | 説明                                                                                           |
| --- | ---------------------------------------------------------------------------------------------- |
| 色  | 背景色。通常=自動運転 · 黄=重要な判断 · 橙=要注意/異常系 · 青=手動確認（表の警告行の色とは別） |

schema: `table-10col-v1` · コード: `TABLE_HEADERS_10` · 読込時 `migrateDocToV2`

### 9 列（段 + 列 · 後方互換）

| 列         | 説明                                                                                 |
| ---------- | ------------------------------------------------------------------------------------ |
| ID         | ノード番号（10, 20…）。他行の接続先にも使う                                          |
| 図形種別   | 端子=開始/終了 · 処理=自動ステップ · 判断=分岐 · 入出力=PLC/上位 · 手動入力=オペ操作 |
| 接続先(下) | 下方向の矢印の行き先 ID                                                              |
| 接続先(右) | 右方向の矢印の行き先 ID（判断の No）                                                 |
| **段**     | 縦位置（同じ段 = 同じ高さで横並び）                                                  |
| **列**     | 横位置（0=左、1=右 … 分岐の並び）                                                    |
| Text1      | 図形に表示する主テキスト（例: MR100 · 取付経路A）                                    |

| Text3      | 補足（3行目）                                                                        |

Excel の **行 / 列** 列は **段 / 列** に相当（取込時は列順で読む · ヘッダー行はスキップ）。

### 8 列（レガシー）

| 列       | 説明                             |
| -------- | -------------------------------- |
| Level    | 同じ行内の横位置（0=左、1=右）   |
| Text1〜3 | 図形に表示するテキスト（複数行） |

## 2. 図形種別 — 使い分け

**結論:** 図形5種＋色4種は現状維持で十分。種類追加より**使い分けルール**を守る。

| 図形種別     | 標準上の意味 | 自動機フローでの用途                                   | 例（Text1）                   |
| ------------ | ------------ | ------------------------------------------------------ | ----------------------------- |
| **端子**     | 開始・終了   | 運転開始 / サイクル完了 / 異常終了                     | `運転開始` · `サイクル完了`   |
| **処理**     | 演算・動作   | シリンダ動作 · タイマ待ち · データ書込                 | `シリンダ前進` · `5s待ち`     |
| **判断**     | 条件分岐     | センサON · 位置完了 · レシピ分岐（**Yes=下 · No=右**） | `原点完了?` · `レシピA?`      |
| **入出力**   | データ入出力 | PLC↔上位 · レシピ読込 · ログ出力                       | `レシピ読込` · `完了信号出力` |
| **手動入力** | 人による操作 | 部品セット · 確認ボタン · メンテ                       | `部品セット` · `確認ボタン`   |

**記号を増やさず書く**

- **待ち・タイマ** → **処理** ＋ Text1（例: `3s待ち`）
- **初期化** → **処理** ＋ Text1（例: `初期化`）
- **共通シーケンス呼び出し** → 当面 **処理** ＋ Text1（例: `共通:原点復帰`）

## 3. 色列 — 使い分け（凡例）

JIS/ISO にフローチャートの色の統一規格はない。3〜4色＋凡例＋形・ラベル併用がベストプラクティス。

| 色（セル値）   | 意味             | 自動機フローでの使い方                                                                   |
| -------------- | ---------------- | ---------------------------------------------------------------------------------------- |
| **（空）通常** | 標準ステップ     | **自動運転の大部分**（全体の9割以上を目安）                                              |
| **黄**         | 判断・注意の強調 | **重要な分岐・インターロック条件**の判断菱形。Text1 に条件を必ず書く                     |
| **橙**         | 要注意・異常系   | **異常復帰・要注意・解除が必要な工程**。Text1 に理由を書く。**表のエラー行（赤）とは別** |
| **青**         | 手動・確認       | **オペ確認・手動介入・セットアップ**。図形種別「手動入力」と併用推奨                     |

**意図的に入れない色**

| 色  | 理由                                                    |
| --- | ------------------------------------------------------- |
| 赤  | 表 UI のエラー行（赤 ring）と混同。HMI でもアラーム専用 |
| 緑  | 色覚多様性。正常/異常を赤緑だけで伝えない               |

### データ源（技術メモ）

| 項目       | 内容                                                                                                                                                                                                |
| ---------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **正本**   | **v2:** `FlowchartDocument.table` の列 index **2**。 **v1 読込:** index 9 → v2 へマイグレーション — [ADR-016](<./意思決定記録(ADR).md#adr-016-10-列順-v2--3-ペインリサイズ2026-06-25>)              |
| **セル値** | 日本語キーワード 4種 + 空。未知値は通常扱い + 警告                                                                                                                                                  |
| **入力**   | 表 UI はセレクト（空 · 黄 · 橙 · 青）— 自由テキスト不可。**Web UI 表示（§E 合意済）:** 空値のラベルは **「-」**（セル値・正規化は空のまま · [表ペインUX方針 §E](../02_機能設計/表ペインUX方針.md)） |
| **凡例**   | 表の列ヘルプ · 本ガイド §3（プレビューへの表示は廃止）                                                                                                                                              |

**やらない:** 自由入力 HEX/RGB · 10列目で図形種別上書き · エッジ色の列制御

## 4. Yes / No とループ（上へ戻る）

| 分岐    | 表の列         | 図の向き                |
| ------- | -------------- | ----------------------- |
| **Yes** | **接続先(下)** | 下へ進む                |
| **No**  | **接続先(右)** | 右へ出る（ラベルは No） |

**上へ戻るループ（推奨パターン）**

過去の ID を **接続先(右) に直接書かない**。右列（**同じ段 · 列=1**）に **処理** 1 行を置き、そこから **接続先(下)** で戻す。

**正本サンプル:** `frontend/src/samples/sample-curry.json` · E2E `e2e/curry-loop.spec.ts`

## 5. 二重コーディング（推奨）

色だけに頼らない（WCAG · High Performance HMI と同趣旨）。

| 伝えたいこと       | 図形         | 色         | ラベル              |
| ------------------ | ------------ | ---------- | ------------------- |
| 重要な分岐         | **判断**     | 黄（任意） | 条件を Text1 に明記 |
| 人の操作           | **手動入力** | 青（推奨） | 作業内容を Text1 に |
| 要注意工程         | **処理** 等  | 橙         | 理由を Text1 に     |
| 通常の自動ステップ | **処理** 等  | （空）     | 動作名を Text1 に   |

---

## 6. やること / やらないこと

**やること**

1. 色は **全体の1割未満** に留める
2. 色の意味は表の列ヘルプ · **本ガイド §3** を参照する
3. 色変更後は **再生成** でプレビューに反映
4. 「5s待ち」「原点復帰」など **特殊動作は Text1 に明記**

**やらないこと**

- ステップごとに意味のないカラフル化
- 赤＝エラー · 緑＝OK だけの配色
- 色だけで異常/正常を区別（ラベルなし）
- 表バリデーションの赤/amber と **色列** を混同しない

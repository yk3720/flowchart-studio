## 1. 入力: 表形式

### 1.1 列定義（8列版 · 後方互換）

> **現行推奨:** §1.3 の **10 列**（段+列+色）。8 列は既存 fixture 互換。

| 列 | 名前 | 型 | 例 |

|----|------|-----|-----|

| 1 | ID | 数値（10刻み推奨） | `10` |

| 2 | 図形種別 | 文字列 | `処理` `判断` `端子` `入出力` `手動入力` |

| 3 | 接続先(下) | カンマ区切り ID | `30,40` |

| 4 | 接続先(右) | カンマ区切り ID | `50` |

| 5 | Level | 整数（0〜） | `0` `1` |

| 6 | Text1 | 文字列 | 主処理 |

| 7 | Text2 | 文字列 | 詳細 |

| 8 | Text3 | 文字列 | 補足 |

### 1.2 簡略版

| 版 | 列数 | 備考 |

|----|------|------|

| 7列版 | 7 | 接続先(右)なし等（仕様書参照） |

| 6列以下 | 6 | 簡易フロー |

パーサは **先頭行の列数** で版を判定する。全行で列数が一致することをバリデーションする。

### 1.3 列定義（9列・10列 · 現行推奨）

[ADR-012](<../03_技術仕様/意思決定記録(ADR).md>) · [作者ガイド.md](./作者ガイド.md)

| 版       | 列数 | レイアウト                              | 備考                                                                                                                                                      |
| -------- | ---- | --------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **10列** | 10   | 段→`tier` · 列→`level` · 色→`colorHint` | **推奨** · `schema: table-10col-v1`（**v2 予定:** [ADR-016](<./意思決定記録(ADR).md#adr-016-10-列順-v2--3-ペインリサイズ2026-06-25>) · `table-10col-v2`） |
| 9列      | 9    | 同上（色列なし）                        | 後方互換                                                                                                                                                  |
| 8列以下  | 6〜8 | 行番号を段の代用 · Level→`level`        | 既存 fixture                                                                                                                                              |

`layoutGrid.ts` は **9 列以上で `tier` を Y 座標の正本**とする（ADR-012 実装済み）。

## 2. 中間表現: FlowNode（IR）

```typescript
/** 1 ノード = 表の 1 行（ID が数値の行） */

export type FlowNode = {
  id: string;

  type: ShapeType;

  fullText: string;

  destsDown: string[];

  destsRight: string[];

  level: number;

  /** 9列+: 段（layoutGrid 段ベース対応時に Y の正本） */

  tier?: number;

  /** 10列: 表「色」列から（空=normal） */

  colorHint?: ColorHint;

  /** 表配列の enumerate インデックス（数値IDでない行もインデックスは進む）— ループ/elbow 判定用 */

  rowIndex: number;
};

export type ShapeType = "端子" | "処理" | "判断" | "入出力" | "手動入力";
```

> 実装 SSOT: `lib/flowchart/model/types.ts`（`ColorHint` は `visual/flowColors.ts`）

### 図形種別の正規化（parse 時）

MZ0000 は部分一致。Web でも同様に正規化してから `ShapeType` に落とす。

| 入力に含む文字列 | 正規化後 |

|------------------|----------|

| `判断` | 判断 |

| `端子` / `開始` / `終了` | 端子 |

| `入出力` / `データ` | 入出力 |

| `手動入力` | 手動入力 |

| 空欄 | 処理 |

| その他 | 処理 |

### その他の正規化

- 接続先: カンマ分割・空白除去・ID 文字列化（`10.0` → `10`）

- Level 空欄 → `0`

- ID が数値でない行 → **スキップ**（行インデックスは進む）

## 4. エッジ: FlowEdge

```typescript
export type ConnectorSite = "top" | "bottom" | "left" | "right";

export type FlowEdge = {
  id: string;

  sourceId: string;

  targetId: string;

  direction: "down" | "right";

  sourceSide: ConnectorSite;

  targetSide: ConnectorSite;

  route: "straight" | "elbow";

  /** 判断ノードのみ。direction から導出（down→Yes, right→No） */

  label?: "Yes" | "No";
};
```

### Yes/No（ADR-003）

- `source` が判断（`type` に `判断` を含む）のときのみ付与

- `direction === "down"` → `"Yes"`

- `direction === "right"` → `"No"`

- ノードの表示テキスト（「Yes側」等）からは **付与しない**

### 接続ルール（概要）

`connector_manager.py` と同型。配置済みノードの `x` で「同一列 straight」判定（`|Δx| < 5`）。

## 5. 永続化: FlowchartDocument

```typescript
export type FlowchartDocument = {
  version: 1;

  /** 例: table-9col-v1 · table-10col-v1（ADR-012 + 色列） */
  schema?: string;

  title?: string;

  table: FlowTableRow[];

  layout: LayoutConfig;

  createdAt: string;
};
```

- **JSON DL/UL:** `FlowchartDocument` 全体

- **localStorage 下書き:** キー `flowchart-studio:draft-v1`（[ADR-009](<../03_技術仕様/意思決定記録(ADR).md>) 実装 · 2026-06-24 に新キーへ書き換え・旧データ破棄 · SSOT: `lib/flowchart/browser/storageKeys.ts`）

- **クラウド（ADR-013 · DB-1）:** Supabase `flow_documents` · `payload jsonb` 列に `FlowchartDocument` 全体。`module_id` は text PK（移行予定）。
- **クラウド（ADR-014 · DB-2 草案）:** 装置構成 **4 テーブル** + `flow_documents.module_id` → `modules.id`（uuid FK）。詳細は [DB設計.md](./DB設計.md)。

- **オフライン閲覧:** IndexedDB 等で直近キャッシュ（ADR-013 · 編集同期は DB-1.5 以降）

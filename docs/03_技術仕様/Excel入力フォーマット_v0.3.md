### 3.1 シート間の役割（決定済み）

**構成メインのハイブリッド** — 編集頻度と SSOT を分ける。

| シート           | SSOT の内容                                     | 作者が見るもの               |
| ---------------- | ----------------------------------------------- | ---------------------------- |
| **`ユニット`**   | ユニットごとの **MID 帯**（枠数 · 開始 · 終了） | 各ユニットに何枠割いたか     |
| **`構成`**       | ユニットに **どの動作を割り当てたか**           | ユニット別の動作一覧（一目） |
| **`モジュール`** | MID の **表示名**（辞書）                       | ラベル定義                   |
| **`U{n}`**       | 各動作の **フロー表**                           | 10 列表データ                |

| 知りたいこと                 | 見る場所                        |
| ---------------------------- | ------------------------------- |
| ユニットに何枠あるか         | `ユニット` の **MID数**         |
| ユニットにどんな動作があるか | **`構成`**（UinID ごとの行）    |
| いま何本割り当て済みか       | **`構成`** の行数（帯内）       |
| 空き番号（欠番）             | 帯内だが **構成に行が無い** MID |

## 4. マスターシート

### 4.1 装置名

| 列       | 例         |
| -------- | ---------- |
| 装置製番 | `A0001`    |
| 装置名   | `塗布装置` |

- データは **2 行目 1 件**（ヘッダー 1 行目）

### 4.2 ユニット

**帯（MID 範囲）の SSOT。** 動作の一覧は載せない（一覧は `構成`）。

| 列          | 意味                                            | 入力             | 例       |
| ----------- | ----------------------------------------------- | ---------------- | -------- |
| ID          | UinID（0 始まり）                               | 手入力           | `0`      |
| ユニット名  | 左ナビのユニット見出し                          | 手入力           | `供給部` |
| **MID数**   | 当該ユニットに割り当てる **枠数**（スロット数） | 手入力           | `16`     |
| **MID開始** | 帯の先頭 MID                                    | 手入力           | `0`      |
| **MID終了** | 帯の末尾 MID                                    | **数式（自動）** | `15`     |

**MID数の意味:** 帯の広さ（最大スロット数）。**構成に載っている動作の本数ではない。**

**MID終了の数式（E 列 · 2 行目以降）:**

```excel
=IF(C2=0,"", D2+C2-1)
```

| MID数    | MID終了                               |
| -------- | ------------------------------------- |
| `0`      | 空（未使用ユニット · プレースホルダ） |
| `1` 以上 | `MID開始 + MID数 - 1`                 |

- **MID終了列は数式固定**（手入力しない）
- 隣接ユニットで帯をつなぐ場合、次行の **MID開始** は `=前行MID終了+1` にしてもよい（任意）。帯の間に隙間を空ける場合は MID開始 を手入力

**A0001 の例:**

| UinID | ユニット名 | MID数 | MID開始 | MID終了 |
| ----- | ---------- | ----- | ------- | ------- |
| 0     | 供給部     | 16    | 0       | 15      |
| 1     | 塗布1部    | 32    | 16      | 47      |
| 2     | 塗布2部    | 32    | 48      | 79      |
| 3     | 塗布3部    | 32    | 80      | 111     |
| 4     | 収納部     | 16    | 112     | 127     |
| 5〜9  | ﾕﾆｯﾄ5 …    | 0     | （空）  | （空）  |

**小規模装置の例:**

| UinID | ユニット  | MID数 | MID開始 | MID終了 |
| ----- | --------- | ----- | ------- | ------- |
| 0     | ユニット1 | 10    | 0       | 9       |
| 1     | ユニット2 | 10    | 10      | 19      |
| 2     | ユニット3 | 10    | 20      | 29      |

帯内の実際の MID は **欠番可**（例: 0, 1, 2, 5 のみ `構成` に載せ、3, 4 は将来用の空き）。

### 4.3 モジュール

| 列   | 意味           | 備考                      |
| ---- | -------------- | ------------------------- | --------------------------------------- |
| ID   | MID            | `0`                       | **一意**。`モジュール` マスター上のキー |
| 動作 | 表示用フル名   | `=CONCATENATE(C{n}&D{n})` | `M` + `ｺﾒﾝﾄ`                            |
| M    | プレフィックス | `="M"&TEXT(A{n},"000")`   |                                         |
| ｺﾒﾝﾄ | 説明テキスト   | 作者手入力                | 予約枠は `XXXX` 等（§11.3）             |

- **欠番:** `モジュール` シートの ID 列は **連番でなくてよい**（帯内の必要な MID だけ行を置く）
- **帯外 MID は禁止** — 正規化時エラー。帯は `ユニット` シートが SSOT（§4.2）

## 5. 構成シート（6 列）

**ユニット ↔ 動作の割当 SSOT。** 作者は主にこのシートで「どのユニットにどんな動作があるか」を編集・一覧する。

### 5.1 ヘッダー

```
装置製番 | 装置名 | UinID | ユニット | MID | モジュール
```

v0.3 判定: 1 行目が `KOSEI_HEADERS_V03` と一致（`python/src/excel_normalize/constants.py`）。

### 5.2 行の意味

- **1 行 = ナビに出す 1 モジュール**（左ナビの 1 葉）
- **行順 = `sort_order`**（上から順にナビ表示）
- **構成に無い MID** = 帯内の欠番 · 将来枠 → **ナビ非表示**（§10.3）
- 各行の MID は、当該 UinID の `[MID開始, MID終了]` 内であること（`ユニット` シートと照合）

### 5.3 数式・入力方針

**新規テンプレで棄却する数式:**

| 棄却                      | 理由                       |
| ------------------------- | -------------------------- |
| `=C2*100+MOD(ROW()-2,10)` | 10 固定 · 旧 per-unit 前提 |
| 全行 `=E{n-1}+1` の連番   | 欠番運用と矛盾             |

**新規の方針:**

- **MID** — 帯内で作者が **明示**（手入力。欠番可）
- **UinID** — `ユニット` の ID と一致。ユニット境界で切り替え
- **ユニット · モジュール** — XLOOKUP でマスター参照可

### 5.4 A0001 の階層サマリー（参考）

| UinID | ユニット | MID数 | MID 範囲 |
| ----- | -------- | ----- | -------- |
| 0     | 供給部   | 16    | 0〜15    |
| 1     | 塗布1部  | 32    | 16〜47   |
| 2     | 塗布2部  | 32    | 48〜79   |
| 3     | 塗布3部  | 32    | 80〜111  |
| 4     | 収納部   | 16    | 112〜127 |

帯の定義は `ユニット` シート（§4.2）。構成行数は割当済み動作数（欠番運用では MID数 以下）。

## 6. ユニットシート（U{n}）

### 6.1 シート名

- **v0.3:** `U{UinID}`（例: 供給部 = `U0`、塗布1部 = `U1`）

## 10. 決定事項（2026-06-26）

### 10.1 MID 採番（Q1）

| 項目     | 決定                                                              |
| -------- | ----------------------------------------------------------------- |
| スキーム | **グローバル一意 MID**（レガシー per-unit ×100 は**採用しない**） |
| 連番     | **不要** — **欠番を許容**                                         |
| 再採番   | **しない** — 帯内の空き番号を将来用に維持                         |
| 一意性   | ファイル内で MID は重複不可                                       |

### 10.2 ユニットあたり可変（Q2）

| 項目 | 決定                                                      |
| ---- | --------------------------------------------------------- |
| 枠数 | ユニットごとに **可変**（`ユニット` の **MID数** で宣言） |
| 帯幅 | 装置・ユニットごとに設計時に決定（10 / 16 / 32 等）       |

### 10.3 空き枠とナビ表示（Q7a）

| 層           | 扱い                             | 左ナビ               |
| ------------ | -------------------------------- | -------------------- |
| `ユニット`   | 帯（枠数）の設計                 | —                    |
| `構成`       | **ナビに出す動作だけ**行を載せる | **載った行のみ**     |
| 帯内の欠番   | `構成` に行なし                  | 非表示               |
| `モジュール` | 名前辞書（任意）                 | 構成に無ければ出ない |
| `U{n}`       | フロー表（記入済み分）           | 構成に無ければ出ない |

**取込・Web（未実装）:** `import.json` / DB の `modules` は **構成の行のみ** upsert。

### 10.4 シート分担 · 帯の入力（Q3b）

| 項目      | 決定                                              |
| --------- | ------------------------------------------------- |
| 帯 SSOT   | **`ユニット` シート** — MID数 · MID開始 · MID終了 |
| 割当 SSOT | **`構成` シート** — UinID + MID + モジュール名    |
| MID終了   | **自動** — `=IF(MID数=0,"", MID開始+MID数-1)`     |
| MID数=0   | 未使用ユニット（プレースホルダ）。帯なし          |

**正規化時の検証（✅ 実装済 · 2026-06-26 · `validate.py` · `inspect.py`）:**

| #    | 検証                                               |
| ---- | -------------------------------------------------- |
| V-B1 | 構成の MID が当該 UinID の `[MID開始, MID終了]` 内 |
| V-B2 | ユニット間で帯が重複しない                         |
| V-B3 | MID数=0 のユニットに構成行が無い                   |
| V-B4 | 構成の MID がファイル内で一意                      |

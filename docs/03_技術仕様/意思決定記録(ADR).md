## ADR-011: Phase 3 画面 — 3 ペイン（装置階層ナビ｜表｜図）

| 項目                         | 内容                                                                                                                                                                                                                                                                                                                                                                                                               |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **状況**                     | 1 装置に多数フロー（ユニット→動作モジュール）があり、現行の 2 ペイン（表｜図）だけでは切替・把握が難しい。フロー作成者は Excel で表を作り取込後に Web で微調整する想定。第 4 回図解 WS（3 ペイン・Pane1 二階層）と grill-me（2026-05-24）で整理。                                                                                                                                                                  |
| **決定**                     | **物理 3 ペイン + ヘッダー**。(1) **AppHeader:** 選択中フロー文脈（ユニット·動作）· 再生成/保存等（**装置名は Nav 最上段** — 2026-05-30 実装）。(2) **Pane1（NavPane）:** **装置セレクタ（最上段）** + ユニット → 動作モジュールの **2 階層**サイドバー。選択単位＝モジュール 1 本＝フロー 1 本。(3) **Pane2（TablePane）:** 8 列表（SoT）· 既存表 UI/CSV/JSON。(4) **Pane3（CanvasPane）:** React Flow 読取専用。 |
| **装置の扱い**               | **2026-05-30 実装:** Nav 最上段 `<select>` で複数装置切替（デモ: プレス機 A/B）。永続化キーは **`${deviceId}:${moduleId}`**（旧キー互換あり）。**ヘッダーに装置名は出さない。**                                                                                                                                                                                                                                    |
| **モジュール切替**           | 切替時に **モジュール ID 単位で表を自動退避・復元**（有力案）。永続化 API → **ADR-013**（Supabase · `moduleDraftRepository` 実装）。DB-1 完了までは mock 差し替え可。                                                                                                                                                                                                                                              |
| **切替時の図**               | 当該モジュールの **前回生成図を表示**。表が最終生成後に変更されていれば **ADR-006/008 と同型の stale**（モジュール単位）。自動再生成はしない。                                                                                                                                                                                                                                                                     |
| **未選択**                   | ユニットのみ展開・モジュール未クリック時は Pane2/3 **空状態**（先頭モジュールの自動選択はしない）。                                                                                                                                                                                                                                                                                                                |
| **レスポンシブ**             | **主:** デスクトップワイドで 3 ペイン横並び。**副:** 狭い画面は Pane1 折りたたみ/ドロワー + Pane2｜Pane3 をタブ「表｜図」（Phase 2 と同型）。                                                                                                                                                                                                                                                                      |
| **理由**                     | 表と図の同時視認・再生成契約（ADR-006/008）を維持しつつ、第 4 回 WS と同様「選ぶ｜編集｜見る」に分割。5 項目を 5 列ペインにすると窮屈。                                                                                                                                                                                                                                                                            |
| **却下**                     | **5 ペイン**（項目ごと 1 列）。**2 ペインにナビを押し込み表｜図タブのみ**（モジュール数が増えると Excel 的シート問題が残る）。**モジュール切替のたび図を空にする**（確認のたび再生成が増える）。**ユニット選択で先頭モジュールを自動選択**（誤編集リスク）。                                                                                                                                                       |
| **Excel（本 ADR では未決）** | 予想: **1 装置 = 1 ファイル · 1 ユニット = 1 シート**。巨大ファイルは **Python 等でフロー用範囲のみ抽出**の可能性。取込 UI 入口（ヘッダー一括 vs Pane2 単体）は実装前に別途合意。                                                                                                                                                                                                                                  |
| **影響**                     | [UI仕様.md](../02_機能設計/UI仕様.md) § Phase 3 を SSOT。**3 ペイン UX ブラッシュアップ実装済み**（2026-05-30 · `FlowchartWorkspace` · `ModuleNavPane`）。クラウド永続化は ADR-013。                                                                                                                                                                                                                               |
| **関連**                     | ADR-006（SoT・stale）· ADR-008（PNG ブロック等）· [テーマ.md](c:/yk-memo/00.ai-driven-school/個人テーマ_フローチャートアプリ/00_テーマ/テーマ.md)（1 装置・PLC 連動）                                                                                                                                                                                                                                              |

## ADR-013: 永続化（Supabase）・認証・公開 — DB-1

| 項目                            | 内容                                                                                                                                                                                                                                                                                                                                                                                               |
| ------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **状況**                        | 編集者 10 人程度＋社内閲覧者で Web 公開したい。現状は localStorage + JSON のみ（ADR-009）。ADR-011 の `moduleDraftRepository` は mock。オフラインでフローが一切見えない状態は避けたい。Neon と Supabase を調査し grill-me（2026-05-27）で整理。                                                                                                                                                    |
| **決定（DB）**                  | **Supabase（Postgres）** に `FlowchartDocument` を **`jsonb`** で保存。初版スキーマは **`profiles`（email · role）+ `flow_documents`（document jsonb · updated_at 等）**。ADR-011 の装置階層（devices / units / modules）は **DB-1 では必須にしない**（`module_id` 等は DB-2 で拡張可）。                                                                                                          |
| **決定（認証・許可）**          | **Supabase Auth** — **Microsoft / Google** の OAuth。`profiles.role` は **`editor` \| `viewer`**。通過は **メール許可リスト**（Table Editor で手動追加）。未登録メールはログイン後も **アクセス拒否**。                                                                                                                                                                                            |
| **決定（役割）**                | **編集者:** 表編集 · 保存 · 再生成 · PNG/SVG/JSON エクスポート。**閲覧者:** 全フロー **閲覧可**（フロー単位の共有指定は不要）· プレビュー + **PNG/SVG のみ** · JSON DL・表編集・保存は不可。                                                                                                                                                                                                       |
| **決定（接続）**                | データ読み書きは **Next.js サーバー（Server Actions / Route Handler）メイン**。サービスロールキーは **ブラウザに出さない**。RLS は **有効化し二重の保険**（`editor` のみ INSERT/UPDATE/DELETE 等）。Middleware のみを認可境界にしない（CVE-2025-29927 等）。                                                                                                                                       |
| **決定（ホスト）**              | アプリは **Vercel**（インターネット到達可 · **ログイン必須**で保護）。Supabase は **本番用 / 開発（プレビュー）用の 2 プロジェクト**。Vercel Production → 本番 DB、Preview → 開発 DB（`VERCEL_RULES` の Preview/本番分離に準拠）。                                                                                                                                                                 |
| **決定（正本・オフライン）**    | オンライン時の正本は **クラウド**。既存 `localStorage`（`flowchart-studio:draft-v1`）は **初回ログイン時の取り込み確認**のみ。オフラインは **閲覧のみ**（IndexedDB 等）— **開いたフローの自動キャッシュ + 「オフライン用に保存」ピン**。表示時は **オフライン・○時点のコピー** を明示。                                                                                                            |
| **着手順**                      | **ADR-011 の 3 ペインより先**に DB-1（ログイン · クラウド保存 · オフライン閲覧キャッシュ）。3 ペイン実装時は `moduleDraftRepository` を Supabase 実装に差し替え。                                                                                                                                                                                                                                  |
| **理由**                        | 10 人運用では端末 localStorage だけでは正本が分散する。Supabase は Auth + Postgres + 管理画面が揃い、許可リスト運用と役割分離を早く載せられる。オフライン編集・同期キューはスコープが別段のため後回し。                                                                                                                                                                                            |
| **却下**                        | **Neon + 自前 Auth のみ**（今回の優先はセットでの立ち上げ速度）。**フロー単位の共有 ACL**（閲覧者は全フローで足りる方針）。**匿名公開編集**。**ブラウザ + anon キーのみで権限を守る**（JSON 制限等はサーバー必須）。**DB-1 からオフライン編集同期**。**Preview が本番 Supabase に書く**構成。                                                                                                      |
| **DB-1 の完了条件（案）**       | (1) editor/viewer でログイン (2) 許可外は拒否 (3) editor が `FlowchartDocument` を保存・読込 (4) viewer は閲覧 + PNG/SVG のみ (5) オフラインで直近キャッシュを表示 (6) 開発/本番 DB 分離                                                                                                                                                                                                           |
| **DB-1.5（本 ADR 外・次候補）** | 編集者の **オフライン編集 + 復帰時同期**（競合方針は別途）。**装置の全モジュールを一括オフライン保存**。                                                                                                                                                                                                                                                                                           |
| **未決**                        | オンライン **同時編集・競合**（最後の保存勝ち / ロック等）。**社内の Supabase・Vercel 利用規定**の正式確認。                                                                                                                                                                                                                                                                                       |
| **実装（2026-05-27）**          | `flowchart-studio`: Supabase Auth · `profiles` / `flow_documents` · Server Actions · IndexedDB オフラインキャッシュ · viewer UI 制限。手順: `docs/runbooks/SUPABASE_SETUP.md`                                                                                                                                                                                                                      |
| **影響**                        | [調査\_永続化とセキュリティ\_Web公開.md](c:/yk-memo/00.ai-driven-school/個人テーマ_フローチャートアプリ/99_アーカイブ/research/調査_永続化とセキュリティ_Web公開.md) · [技術方針.md](../03_技術仕様/技術方針.md) · [データモデル.md](../03_技術仕様/データモデル.md) §5 永続化 · ADR-011 の永続化 API → 本 ADR に接続。実装: `creating-vercel-yk` · `VERCEL_RULES` · Supabase ダッシュボード運用。 |
| **関連**                        | ADR-006/008（SoT・stale・エクスポート契約）· ADR-009（localStorage キー互換）· ADR-011（モジュール単位退避の先送りスキーマ）                                                                                                                                                                                                                                                                       |

## ADR-015: フロー共同編集 — 編集は全 editor · 削除は所有者/admin（2026-06-15）

| 項目     | 内容                                                                                                                                                                                                               |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **状況** | DB-2 適用後、grill-me（2026-06-12）で装置/ユニット/動作の**削除権限**を所有者チェーンで実装（010/011/013）。015 では一時的に「フロー編集も作成者のみ」としたが、社内運用では**他人フローの表編集・保存**が必要。   |
| **決定** | **フロー中身の編集**（クラウド保存 · import 再取込）は **全 `editor` / `admin`** が可能（共同編集）。**削除系**（動作削除 · 雛形リセット · 装置/ユニット削除）は **従来どおり** 所有者チェーン + `admin` のみ。    |
| **理由** | 社内では複数 editor が同一装置のフローを分担編集する。誤削除の方がデータ損失リスクが大きいため、削除だけ所有者制限を維持。                                                                                         |
| **実装** | `016_flow_documents_collaborative_edit.sql`（**正本**）。`015` は `016` で上書き — 015 未適用なら 016 のみ Run。                                                                                                   |
| **却下** | 015 の「編集も flow 作成者のみ」（共同編集運用と矛盾）· 削除まで全 editor 開放                                                                                                                                     |
| **未決** | **同時編集の競合**（最後の保存勝ち / ロック / OT 等）は本 ADR 範囲外 — ADR-013 の未決を継続                                                                                                                        |
| **状態** | **Accepted** · 本番 DB 適用済（2026-06-15）· アプリ push 済                                                                                                                                                        |
| **関連** | ADR-013（認可モデル）· ADR-014（削除方針）· [`SUPABASE_RULES.md`](c:/yk-skill/rule/30_web_stack/SUPABASE_RULES.md) §3 · [`HANDOFF.md`](c:/yk-memo/handoffs/flowchart-studio/HANDOFF.md) §6 権限表 · Runbook §12-2c |

## ADR-016: 10 列順 v2 · 3 ペインリサイズ（2026-06-25）

| 項目                                | 内容                                                                                                                                                                                                                                                                                                                      |
| ----------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **状況**                            | 10 列はレガシー 8 列に段/列/色を後付けした順序のまま。色列が index 9 で図形種別から離れ作者の二重コーディング動線が悪い。接続先列ヘッダーが数字1桁入力に対し過剰な幅。ペイン幅は CSS 固定で表を広げられない。                                                                                                             |

| **決定（v2 列順 · index 0-based）** | `ID` · `図形種別` · `色` · `接続先(下)` · `接続先(右)` · `段` · `列` · `Text1` · `Text2` · `Text3`                                                                                                                                                                                                                        |

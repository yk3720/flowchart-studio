## 3. 決まったこと

### 3.1 優先順位

| 優先         | 内容                                                                        |
| ------------ | --------------------------------------------------------------------------- |
| **最優先**   | フローチャートを**素早く作れる**こと。ここに手間がかかればプロジェクト失敗  |
| **次点**     | 作ったフローを**簡単に見つけて開ける**こと（DB 保存は重要）                 |
| **後回し可** | 見た目（テーマ・サイズ等）· 年間テーマの本番投入 · ログイン整備（OAuth 等） |

### 3.2 検証の方向（方式 C）

- **Web 版で本当にいいか** · **React Flow でいいか**を、触りながら検証したい
- 年間テーマは急がない。**方向性を決める時間**を使う
- 上司・同僚に見せる前に、**画面・機能をブラッシュアップ**したい

### 3.3 7月前後の「見せ方」

| 項目         | 内容                                                                  |
| ------------ | --------------------------------------------------------------------- |
| 時期         | **7月くらい**に見てもらえればよい（**必須ではない** · 8月以降でも可） |
| 形式         | **気軽な相談**（最終発表ではない）                                    |
| 手段         | 自分の PC（`npm run dev`）でも本番 URL でも可。ログイン整備は後でよい |
| 見せたい操作 | **作る**（表編集 → 再生成 → 図）＋ **探して開く**（保存済みフロー）   |
| 優先         | 迷ったら **作る操作**                                                 |

### 3.4 入力・作成の想定

| 項目                             | 内容                                                                                                                    |
| -------------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| 表の作成場所                     | **Excel またはスプレッドシート**                                                                                        |
| 取込方法                         | **Excel からコピーして貼り付け** · **Excel ファイルを選んで読み込み**                                                   |
| 7月までの Excel 機能             | **今ある機能を分かりやすくする**のが第一優先（新規開発より UI 整理）                                                    |
| プレビュー縦最大化（2026-05-31） | フロー図エリアを**画面上下いっぱい**に — 図上の dead space 削除（[計画 §D](./計画_2026-05-30_図の見た目と装置階層.md)） |

**補足（実装現状 · 2026-05-30）:** `CsvPastePanel` に貼り付けと「Excel ファイル…」あり。シートは自動選択。範囲指定は未実装。

### 3.5 画面整理

- **いらないボタン・表示を減らしてすっきり**させたい
- **何を消すかは改めて本人が指定**する（本相談では具体リストは未確定）
- 相談しながら消す → 自分で使って修正、という進め方

### 3.6 年度末の本当の目標

- **2026 年度中**に、**装置 1 件分のデータが入力できる体制**ができていれば OK
- 7月の見せ方は上記の「できれば」

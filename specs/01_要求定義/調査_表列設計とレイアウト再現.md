# 調査 — 表列設計と Excel 手調整レイアウトの再現

**調査日:** 2026-05-24  
**きっかけ:** M002（供給*SUS板搬送*取）を Web 取込したが、Excel 手調整図と見た目が大きく異なる  
**結論:** 8列固定は不要。**段 + 列** を明示する 9〜10 列モデル + レイアウト算法変更で、横並び分岐まで表だけで再現可能（合流バス・色は別途）

---

## 1. 本調査で確定したこと

### 1-1. Excel 取込（現行 8 列）

| 方法                           | 可否           | 備考                               |
| ------------------------------ | -------------- | ---------------------------------- |
| コピー＆ペースト（タブ区切り） | **可**         | テキストエリア →「貼り付けを反映」 |
| `.xlsx` / `.xls` ファイル      | **可**         | SheetJS · シート名「表」等を優先   |
| 反映後                         | **再生成**必須 | プレビューは表編集と stale 契約    |

実装: `flowchart-studio` — `CsvPastePanel.tsx` · `parseCsv.ts` · `parseExcel.ts`

### 1-2. Excel 手調整図を「自動で」再現できるか

| 質問                          | 答え                                                     |
| ----------------------------- | -------------------------------------------------------- |
| 今のアプリのまま              | **No** — 表行番号 = 縦位置 · Level = その行内の横位置    |
| 表の入力方法を変え + 算法改善 | **Yes（条件付き）** — 段・列を明示する列モデルが最も素直 |
| 図形ドラッグ微調整            | **意図的に非対応**（ADR-010）                            |

### 1-3. M002 でズレる主因

- ID 3・4・5（並列分岐）が **別表行** + Level 1,2,3 → Web では **段差（斜め段）** になる
- Excel 手調整は **同じ高さで横 3 列** + 合流バス
- 接続先（`3,4,5` · 合流 `6`）は **正しくパースされる** — 問題は座標モデル

---

## 2. 現行 Web の座標モデル（8 列）

```
表の行番号 (rowIndex) → Y（上から 1 段ずつ）
Level（列5）          → X（その段内の横オフセット）
```

コード: `parseTable.ts` → `layoutGrid.ts`（`c:/yk-application/flowchart-studio/lib/flowchart/`）

**Level のヘルプ:**「同じ行内の横位置」— 並列ノードが別表行にあると意味が破綻する。

## 3. 業界調査 — 表駆動フローの 3 系統

### 系統 A：接続中心（Visio Data Visualizer 型）

| 列                       | 役割                                             |
| ------------------------ | ------------------------------------------------ |
| Process Step ID          | 一意 ID                                          |
| Description              | 表示テキスト                                     |
| **Next Step ID**         | 次 ID（分岐 `3,4,5` カンマ区切り・スペースなし） |
| Connector Label          | 矢印ラベル（`Yes,No`）                           |
| Shape Type               | 端子・処理・判断…                                |
| Phase / Function（任意） | 段 · スイムレーン                                |

- **座標列なし** — レイアウトは Visio 内部エンジン
- 参照: [Microsoft Visio Data Visualizer](https://support.microsoft.com/en-us/office/create-a-diagram-in-excel-with-the-visio-data-visualizer-add-in-bee3b5aa-aaaf-4401-acc6-276b711c763c) · [Clovis Research 形式解説](https://www.clovisresearch.com/visio-data-visualizer/data-visualizer-dataset-format/)

### 系統 B：グリッド明示（yFiles LayoutGrid 型）— **本テーマ第一候補**

| 列                    | 役割                      |
| --------------------- | ------------------------- |
| ID · 種別 · 接続先    | 現行と同型                |
| **段（Row / Tier）**  | 縦位置（同値 = 同じ高さ） |
| **列（Col / Lane）**  | 横位置（0=左）            |
| Text1〜3 · 色（任意） | 表示 · ハイライト         |

- 参照: [yFiles LayoutGrid](https://docs.yfiles.com/yfiles-html/dguide/layout-table_layout/) · [yEd node rank](https://yed.yworks.com/support/manual/layout/layout_hierarchic_incremental.html)

### 系統 C：ハイブリッド

- 接続先から自動推定 + 段・列で上書き
- PLC 自動変換の最終形向き · POC 初期は B が早い

### その他参考

| ソース                                                                                                     | メモ                                    |

## 4. 推奨 — 9〜10 列モデル（段 + 列）

### 4-1. 列定義案

```
ID | 図形種別 | 接続先(下) | 接続先(右) | 段 | 列 | Text1 | Text2 | Text3 | 色(任意)
```

| 変更           | 内容                               |
| -------------- | ---------------------------------- |
| **段**（新設） | 縦位置の正本（表行順は編集用のみ） |
| **列**         | 現 Level をリネーム · 横位置専用   |
| **色**（任意） | 黄 · 緑など（M002 のハイライト）   |

### 4-2. M002 の段・列付け例

| ID  | 接続先(下) | 段     | 列    | 備考     |
| --- | ---------- | ------ | ----- | -------- |
| 1   | 2          | 1      | 0     | 開始     |
| 2   | 3,4,5      | 2      | 0     | 分岐元   |
| 3   | 6          | **3**  | **0** | 並列左   |
| 4   | 6          | **3**  | **1** | 並列中   |
| 5   | 6          | **3**  | **2** | 並列右   |
| 6   | 7          | 4      | 0     | 合流後   |
| 11  | 12,13      | 10     | 0     | 2 分岐元 |
| 12  | 14         | **11** | **0** |          |
| 13  | 14         | **11** | **1** |          |
| 14  |            | 12     | 0     | 終了     |

### 4-3. 算法変更（実装側）

1. `layoutGrid` — Y を **表行番号** ではなく **段** で決定
2. X を **列 × (width + gapH)** で決定
3. **合流バス** — 矢印描画の改善（列だけでは不足）
4. 8 列 Excel の後方互換 — 段=行番号 · 列=Level として読む fallback（移行期）

### 4-4. 却下・保留

| 案                   | 理由                                                             |
| -------------------- | ---------------------------------------------------------------- |
| 8列の Level だけ直す | 同じ高さの並列を表現できない                                     |
| Visio 型 + ELK のみ  | ADR「dagre でノード位置を決めない」と衝突しうる · 合流バスは別途 |
| 図形ドラッグ         | ADR-010 で見送り                                                 |

## 5. 表だけでは再現しにくい要素

| 要素                 | 対応             |
| -------------------- | ---------------- |
| 横並び分岐           | **段 + 列** で可 |
| 合流の横バス（幹線） | エッジ描画算法   |
| ノード色             | **色** 列        |
| ピクセル微調整       | 非スコープ       |

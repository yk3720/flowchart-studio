# リポジトリ構造 — flowchart-studio

**用途:** 独立アプリリポの **フォルダ構成 SSOT**（YK テンプレート実例 · `yk-application`）。

**最終更新:** 2026-06-24（repo-layout v2 — 系統別整理）

---

## 設計原則

| 層                 | フォルダ                        | 載せるもの                             |
| ------------------ | ------------------------------- | -------------------------------------- |
| **憲法**           | ルート `AGENTS.md`              | AI 境界 · 再開手順 · SSOT マップ       |
| **Product Spec**   | `specs/`                        | 何を作るか（要求〜ADR · decision-log） |
| **運用**           | `docs/`                         | どう動かすか（LOCAL_DEV · Runbook）    |
| **実装**           | `app/` · `components/` · `lib/` | Next.js 本線                           |
| **サンプルデータ** | `samples/`                      | アプリ用 JSON（雛形・デモ表）          |
| **自動化**         | `scripts/`                      | seed · Supabase SQL · 起動 bat         |
| **補助ツール**     | `tools/`                        | 独立パッケージ（例: excel_normalize）  |
| **DB スキーマ**    | `supabase/migrations/`          | マイグレーションのみ                   |
| **E2E**            | `e2e/`                          | Playwright                             |
| **セッション**     | `yk-memo/handoffs/{slug}/`      | 次の 1 件（**リポ外**）                |

**MUST NOT:** 仕様 stub を `docs/` に置く · `fixtures` という汎用名を複数系統で使う · リダイレクト用 MD を増やす。

## ツリー（正本）

```text
flowchart-studio/
├── AGENTS.md                 # エージェント憲法（唯一の入口）
├── README.md
├── CHANGELOG.md
├── package.json              # + next/eslint/playwright 等の設定（ルート慣習）
├── フローチャートを開く.bat    # → scripts/dev/open.bat へ委譲
│
├── specs/                    # Product Spec（コードと同居）
│   ├── 01_要求定義/
│   ├── 02_機能設計/
│   ├── 03_技術仕様/          # ADR 正本
│   ├── 04_リポジトリ構造/    # 本ファイル
│   ├── 05_開発ガイドライン/  # decision-log
│   └── 06_ユビキタス言語/
│
├── docs/                     # 運用のみ
│   ├── README.md
│   ├── LOCAL_DEV.md
│   ├── design-system.md
│   └── runbooks/
│       ├── SUPABASE_SETUP.md
│       └── DB2_MIGRATION_RUNBOOK.md
│
├── samples/                  # アプリ用サンプル JSON
├── scripts/
│   ├── dev/open.bat
│   ├── seed/                 # TS: DB seed
│   └── supabase/             # SQL: 関数 · 生成 Run 用
├── tools/
│   └── excel_normalize/      # Python 取込パイプライン
│       ├── testdata/         # 装置 Excel · import.json
│       └── scripts/
├── supabase/migrations/
├── app/
├── components/
├── lib/
└── e2e/
```
